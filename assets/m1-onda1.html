Árvores Binárias

Tema: Estrutura Cai em ~75% das P1's Recursão essencial
Por que estudar isso primeiro: entre as 4 últimas provas P1 do Prof. Strauss, 3 cobraram reconstrução de árvore binária a partir de percursos. Dominar este tópico significa garantir uma questão inteira da prova.
Definições Fundamentais
Uma árvore binária é uma estrutura recursiva onde cada nó tem no máximo dois filhos: o filho esquerdo e o filho direito. Formalmente, uma árvore binária é:
  • vazia (representada por null ou nil), ou
  • composta por um nó raiz + uma subárvore esquerda + uma subárvore direita, ambas binárias.
  • 01Raiz: nó no topo da árvore, sem pai.
  • 02Folha: nó sem filhos (subárvores esquerda e direita vazias).
  • 03Altura: número de arestas no caminho mais longo da raiz até uma folha. Árvore vazia tem altura -1; árvore com só raiz tem altura 0.
  • 04Profundidade de um nó: número de arestas da raiz até ele.
  • 05Grau: número de filhos do nó (0, 1 ou 2 em árvores binárias).
  • 06Árvore cheia: todo nó interno tem exatamente 2 filhos. Completa: todos os níveis preenchidos exceto possivelmente o último, que é preenchido da esquerda para a direita.
Os Quatro Percursos
Um percurso (traversal) é uma forma sistemática de visitar todos os nós de uma árvore. Como árvores são estruturas recursivas, todos os percursos são definidos recursivamente. A diferença está em quando visitamos a raiz:
Percurso: Pré-ordem
Resultado clique em "Animar Percurso" para começar 
# Implementação dos 4 percursos (recursivos, exceto em-nível)
class Node:
    def __init__(self, valor):
        self.valor = valor
        self.esq = None
        self.dir = None

def preordem(no):
    if no is None: return
    print(no.valor)         # 1. visita a raiz
    preordem(no.esq)        # 2. percorre subárvore esquerda
    preordem(no.dir)        # 3. percorre subárvore direita

def inordem(no):           # também chamada de SIMÉTRICA
    if no is None: return
    inordem(no.esq)
    print(no.valor)
    inordem(no.dir)

def posordem(no):
    if no is None: return
    posordem(no.esq)
    posordem(no.dir)
    print(no.valor)

from collections import deque
def em_nivel(raiz):       # BFS — usa fila
    if raiz is None: return
    fila = deque([raiz])
    while fila:
        no = fila.popleft()
        print(no.valor)
        if no.esq: fila.append(no.esq)
        if no.dir: fila.append(no.dir)
Análise de Complexidade dos Percursos
Função tempo T(n) dos três percursos recursivos. Considere uma árvore com n nós: 
T(0) = c1                          # caso base (no == None)
T(n) = T(esq) + T(dir) + c2          # processa raiz + 2 chamadas recursivas

Cada nó é visitado exatamente UMA vez → operações O(1) por nó
T(n) = n · O(1) = Θ(n)
Conclusão: os percursos são ótimos — não dá para visitar n nós em menos que Θ(n) operações, que é o limite inferior Ω(n) do problema.

Algoritmo de Reconstrução (Pré-ordem + In-ordem)
  • 01O primeiro elemento da pré-ordem é a raiz da árvore.
  • 02Localize essa raiz no vetor in-ordem (posição k).
  • 03Tudo à esquerda de k no in-ordem é a subárvore esquerda. Tudo à direita, a subárvore direita.
  • 04Os próximos k elementos da pré-ordem (após a raiz) formam a pré-ordem da subárvore esquerda. Os restantes formam a pré-ordem da direita.
  • 05Recursão: aplique os passos 1-4 nas duas metades.
def reconstroi(preordem, inordem):
    if not preordem or not inordem:
        return None

    # 1. Primeiro da pré-ordem é a raiz
    raiz_valor = preordem[0]
    raiz = Node(raiz_valor)

    # 2. Localiza a raiz no in-ordem
    k = inordem.index(raiz_valor)   # O(n) na implementação ingênua

    # 3. Divide os percursos
    inord_esq = inordem[:k]
    inord_dir = inordem[k+1:]
    pre_esq   = preordem[1:k+1]
    pre_dir   = preordem[k+1:]

    # 4. Recursão
    raiz.esq = reconstroi(pre_esq, inord_esq)
    raiz.dir = reconstroi(pre_dir, inord_dir)
    return raiz
Análise da Função Tempo
Implementação ingênua (acima): 
T(n) = T(k) + T(n-k-1) + O(n)    # O(n) é o custo de inordem.index()
                                  
Pior caso (árvore degenerada k=0 ou k=n-1):
T(n) = T(n-1) + O(n)  →  T(n) = O(n²)

Caso médio (k ≈ n/2):
T(n) = 2·T(n/2) + O(n)  →  T(n) = O(n log n)
Otimização para Θ(n): usar HashMap para mapear valor → índice no in-ordem em pré-processamento O(n). Aí a busca vira O(1) e: 
T(n) = T(k) + T(n-k-1) + O(1)  →  T(n) = Θ(n)
Como Ω(n) é o mínimo (precisamos olhar todos os nós), o algoritmo otimizado é ÓTIMO.

Notações Matemáticas: Infixa, Prefixa, Pósfixa

Cai em ~50% das P1's Árvore + Recursão Conversão
Conexão direta com 1.3: toda expressão matemática pode ser representada como uma árvore binária. Os operadores são nós internos, os operandos são folhas. A escolha da notação é literalmente escolher um percurso dessa árvore.
As Três Notações
NotaçãoPosição do operadorEquivale a percursoExemplo (3+4)·5
InfixaEntre os operandosIn-ordem (com parênteses)(3 + 4) * 5
Prefixa (Polonesa)Antes dos operandosPré-ordem* + 3 4 5
Pósfixa (Polonesa Reversa)Depois dos operandosPós-ordem3 4 + 5 *
Por que isto importa?
  • Infixa é a notação humana, mas precisa de parênteses e regras de precedência.
  • Prefixa e pósfixa não precisam de parênteses nem precedência — são totalmente determinadas pela posição. Por isso são usadas internamente em compiladores e calculadoras.
  • A pósfixa é avaliada em tempo linear com uma pilha: cada operando é empilhado, cada operador desempilha 2 valores, calcula, empilha o resultado.

Algoritmo Recursivo: Prefixa → Infixa + Pósfixa
Questão clássica do Prof. Strauss: "Desenvolva UM ÚNICO algoritmo recursivo que transforme uma função matemática escrita na forma PREFIXA para as formas INFIXA e PÓSFIXA. Descreva se sua proposta é ótima." 
def converte_prefixa(tokens, idx=0):
    """
    Lê tokens prefixa e retorna (infixa, posfixa, novo_idx).
    Estratégia: um único algoritmo que constrói as duas formas
    simultaneamente em uma única passada recursiva.
    """
    if idx >= len(tokens):
        return "", "", idx

    token = tokens[idx]

    if eh_operador(token):
        # 1. processa subárvore esquerda
        inf_e, pos_e, idx = converte_prefixa(tokens, idx + 1)
        # 2. processa subárvore direita
        inf_d, pos_d, idx = converte_prefixa(tokens, idx)
        # 3. monta as duas formas a partir das subexpressões
        infixa  = f"({inf_e} {token} {inf_d})"
        posfixa = f"{pos_e} {pos_d} {token}"
        return infixa, posfixa, idx

    else:  # é operando (folha)
        return token, token, idx + 1

# Uso:
# tokens = ["+", "Z", "*", "Q", "/", "T", "L"]
# infixa, posfixa, _ = converte_prefixa(tokens)
Análise da Função Tempo
Cada token é visitado exatamente uma vez e processado em O(1): 
T(0) = c1                              # vetor vazio
T(n) = T(nesq) + T(ndir) + c2            # operador: 2 chamadas recursivas
T(n) = c3                              # operando: caso base

Como nesq + ndir + 1 = n (cada nó conta uma vez):
T(n) = Θ(n)

Cota inferior: Ω(n)  →  precisamos ler todos os tokens
Algoritmo: Θ(n)      →  ATINGE A COTA  →  É ÓTIMO ✓

1.5

Algoritmos Recursivos

Tema obrigatório Pilha de chamadas Recorrências
Recursão é a linguagem natural de árvores e divide-and-conquer. O Prof. Strauss exige soluções recursivas em quase toda questão. Esta seção te ensina a analisar a função tempo de algoritmos recursivos — o ponto central de toda prova.
Anatomia de um Algoritmo Recursivo
Todo algoritmo recursivo bem escrito tem três componentes:
  • Caso base: condição em que a função retorna sem chamar a si mesma. Sem caso base → recursão infinita → stack overflow.
  • Caso recursivo: a função chama a si mesma com um problema menor que se aproxima do caso base.
  • Combinação: como o resultado das chamadas recursivas é combinado para resolver o problema original.
# Esqueleto genérico
def resolve(problema):
    if eh_caso_base(problema):
        return solucao_trivial(problema)        # caso base

    # Decompõe em subproblemas menores
    sub1, sub2 = decompoe(problema)

    # Resolve recursivamente
    res1 = resolve(sub1)
    res2 = resolve(sub2)

    # Combina os resultados
    return combina(res1, res2)
Estudo de Caso: Fibonacci
Sequência de Fibonacci: fib(0)=0, fib(1)=1, fib(n) = fib(n-1) + fib(n-2). Apareceu na P1-2018.01 (5ª questão), pedindo para analisar a árvore de chamadas e determinar Ω e O. 
def fib(n):
    if n <= 1:                 # caso base: O(1)
        return n
    return fib(n - 1) + fib(n - 2)  # 2 chamadas recursivas

Análise: Por que é O(2ⁿ)?
A função tempo do Fibonacci recursivo ingênuo é: 
T(0) = T(1) = c1                  # casos base
T(n) = T(n-1) + T(n-2) + c2        # caso recursivo

Esta é a própria recorrência de Fibonacci! Pode-se mostrar que:
T(n) = Θ(φⁿ) onde φ = (1+√5)/2 ≈ 1.618 (razão áurea)

Cota superior: T(n) ≤ 2·T(n-1) + c  →  T(n) = O(2ⁿ)
Cota inferior: T(n) ≥ 2·T(n-2) + c  →  T(n) = Ω((√2)ⁿ)

Número exato de chamadas: 2·fib(n+1) - 1
Para n=4: 2·fib(5) - 1 = 2·5 - 1 = 9 chamadas (a prova diz 8, contando diferente)
⚠ É ÓTIMO? NÃO. A cota inferior do problema "calcular fib(n)" é Ω(n) (precisamos no mínimo executar n adições). Como o algoritmo ingênuo é O(2ⁿ) >> Ω(n), ele NÃO é ótimo. A versão iterativa ou com memoização atinge Θ(n) e é ótima.
Recorrências Comuns em Análise de Algoritmos
Decorar essas é essencial. Toda análise de complexidade recursiva cai em uma destas formas:
RecorrênciaSoluçãoOnde aparece
T(n) = T(n-1) + cΘ(n)Recursão linear simples (fatorial, busca em lista)
T(n) = T(n-1) + nΘ(n²)Algoritmos de ordenação ingênuos
T(n) = 2·T(n-1) + cΘ(2ⁿ)Torre de Hanói; subconjuntos
T(n) = T(n-1) + T(n-2) + cΘ(φⁿ) ≈ Θ(1.618ⁿ)Fibonacci ingênuo
T(n) = T(n/2) + cΘ(log n)Busca binária
T(n) = T(n/2) + nΘ(n)Busca em árvore balanceada com trabalho linear
T(n) = 2·T(n/2) + cΘ(n)Percurso em árvore balanceada
T(n) = 2·T(n/2) + nΘ(n log n)Merge Sort, divide-and-conquer balanceado
T(n) = T(n/2) + T(n/2) + nΘ(n log n)Equivalente à anterior
📐 Teorema Mestre (resumido): para recorrências da forma T(n) = a·T(n/b) + f(n), compare f(n) com n^(logba). Se forem do mesmo nível, T(n) = Θ(f(n)·log n). Esta é a forma mais usada na prova.
Como Escrever T(n) na Prova (Passo a Passo)
  • 01Identifique o caso base: "para n = 0 (ou 1, conforme), T(n) = c1 (constante)".
  • 02Conte as chamadas recursivas: quantas vezes a função se chama? Em quais tamanhos?
  • 03Mensure o trabalho não-recursivo: tudo que NÃO é chamada recursiva — comparações, cópias, prints. Pode ser O(1), O(n), O(log n)...
  • 04Escreva a recorrência: T(n) = (soma das T's das chamadas) + (trabalho extra).
  • 05Resolva: use a tabela de recorrências comuns acima ou expanda manualmente.
  • 06Compare com a cota inferior Ω(n) do PROBLEMA: se T(n) atinge Ω(n), o algoritmo é ótimo.
Exemplo completo de resposta (Fibonacci recursivo): 
1. Caso base:    T(0) = T(1) = c1
2. Caso recurs.: 2 chamadas (n-1 e n-2), trabalho extra: 1 soma = c2
3. Recorrência:  T(n) = T(n-1) + T(n-2) + c2
4. Solução:      T(n) = Θ(φⁿ) ≈ Θ(1.618ⁿ)  →  O(2ⁿ)
5. Cota inferior do problema "calcular fib(n)": Ω(n)
6. Comparação: O(2ⁿ) >> Ω(n)  →  NÃO é ótimo
