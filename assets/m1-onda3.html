Grafos — Teoria e Algoritmos

Aplicação: Rede e Museu Pt.2 BFS · DFS · Dijkstra Estrutura universal
Onde cai na P1: a P1-2018.01 (Q1) pediu uma estrutura para representar uma "GRID NETWORK" e detectar a "máquina gestora" (nó central). A P1-Strauss (Museu Pt.2) pede a melhor trajetória do ladrão evitando câmeras — clássico problema de caminho mínimo em grafo.
Definições Fundamentais
Um grafo G = (V, E) é uma estrutura formada por:
  • V: conjunto de vértices (nós).
  • E: conjunto de arestas, onde cada aresta conecta um par de vértices.
Tipos de grafos:
  • Direcionado: arestas têm direção (u → v ≠ v → u).
  • Não-direcionado: arestas são bidirecionais.
  • Ponderado: cada aresta tem um peso (ex: distância, custo).
  • Conexo: existe caminho entre todo par de vértices.
  • 01Grau de um vértice: número de arestas incidentes a ele. Em grafos direcionados, há grau de entrada (in-degree) e grau de saída (out-degree).
  • 02Caminho: sequência de vértices onde cada par consecutivo é conectado por aresta. Comprimento = nº de arestas.
  • 03Ciclo: caminho fechado (vértice inicial = final).
  • 04Árvore: grafo conexo, não-direcionado, sem ciclos. Tem exatamente V−1 arestas.
Representações em Memória
Existem duas representações principais. A escolha depende do problema:

Matriz de Adjacência

M[u][v] = 1 se aresta uv existe
# Vetor 2D V × V
adj = [[0] * V for _ in range(V)]
adj[u][v] = 1
adj[v][u] = 1  # não-direcionado

Espaço: Θ(V²) sempre.
Adjacência? O(1).
Vizinhos de u? O(V).
Use quando: grafos densos, ou quando V é pequeno.

Lista de Adjacência

adj[u] = [vizinhos de u]
# Vetor de listas
adj = [[] for _ in range(V)]
adj[u].append(v)
adj[v].append(u)  # não-direcionado

Espaço: Θ(V + E).
Adjacência? O(grau(u)).
Vizinhos de u? O(grau(u)).
Use quando: grafos esparsos (E << V²), ou para BFS/DFS.

Busca em Largura (BFS)
Visita os vértices em ordem de distância a partir da origem. Encontra o caminho com menor número de arestas em grafos não-ponderados. Usa fila (FIFO). 
from collections import deque

def bfs(adj, origem):
    visitados = {origem}
    fila = deque([origem])
    ordem = []
    while fila:
        u = fila.popleft()
        ordem.append(u)
        for v in adj[u]:
            if v not in visitados:
                visitados.add(v)
                fila.append(v)
    return ordem

# Complexidade: cada vértice é visitado 1 vez (V) e cada aresta também (E)
# T(V, E) = Θ(V + E)
Busca em Profundidade (DFS)
Vai o mais fundo possível antes de retroceder. Usa pilha (LIFO) — implementação recursiva é a mais natural. Útil para detectar ciclos, ordenação topológica, componentes conexos. 
def dfs(adj, u, visitados=None, ordem=None):
    if visitados is None: visitados = set()
    if ordem is None: ordem = []
    visitados.add(u)
    ordem.append(u)
    for v in adj[u]:
        if v not in visitados:
            dfs(adj, v, visitados, ordem)
    return ordem

# Complexidade: Θ(V + E) — mesma análise da BFS

Aplicação: Identificar a Máquina Gestora (P1-2018.01 Q1)
Problema: dada uma rede de N máquinas com protocolos GRID e NÃO-GRID, identifique qual é a "máquina gestora" — aquela que conhece TODAS as outras (tem grau = N-1 no subgrafo de comunicações GRID). 
def acha_gestora(adj, n):
    """
    adj: lista de adjacência das comunicações GRID
    Retorna o índice da máquina gestora ou None.
    """
    for u in range(n):
        if len(adj[u]) == n - 1:
            return u
    return None

# Análise:
# T(n) = Σ verifica_grau(u) = Σ O(1) = O(n)
# Cota inferior: Ω(n) — precisamos olhar todas as máquinas
# É ÓTIMO ✓
📐 Estrutura de dados ideal: lista de adjacência, pois grafos de rede são esparsos. Para detectar a gestora basta uma varredura linear nos graus dos vértices — não precisa de DFS/BFS, basta contar conexões.
Caminho Mínimo (Dijkstra) — Aplicação no Museu
Para a Parte 2 do problema do Museu (trajetória do ladrão evitando câmeras), modelamos a sala como um grafo: pontos seguros (não cobertos por câmera) são vértices conectados por arestas com peso = distância. Dijkstra encontra o caminho mais curto. 
import heapq

def dijkstra(adj, origem, n):
    """
    adj: dict {u: [(v, peso), ...]}
    Retorna lista dist[v] = menor distância de origem a v.
    """
    dist = [float('inf')] * n
    dist[origem] = 0
    heap = [(0, origem)]   # (distância, vértice)

    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]: continue   # versão obsoleta
        for v, peso in adj[u]:
            nd = d + peso
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(heap, (nd, v))
    return dist

# Complexidade: O((V + E) log V) com heap binário

Análise das 4 últimas P1's do Prof. Strauss: esta seção condensa o que invariavelmente cai e como o professor avalia. Use como checklist final antes de entregar.
Anatomia da Resposta Ideal
Toda resposta a uma questão da P1 deve seguir esta estrutura. A ordem e completude valem nota.
1
Modelagem do Problema
Descreva em palavras qual é o problema, identifique entradas e saídas. Indique explicitamente qual estrutura de dados você usará e justifique a escolha. Exemplo: "Usarei lista de adjacência porque o grafo é esparso (E << V²) e precisarei iterar pelos vizinhos."
2
Algoritmo (Pseudocódigo ou Python)
Use recursão sempre que possível — Strauss valoriza. Comente o que cada bloco faz. Identifique caso base e caso recursivo. Para problemas iterativos, descreva a invariante do laço.
3
Função Tempo T(n) — OBRIGATÓRIO
Strauss SEMPRE escreve "Lembre-se de obrigatoriamente definir a função tempo!". Escreva a recorrência: T(n) = T(...) + .... Identifique cada termo: caso base, chamadas recursivas, trabalho não-recursivo. Resolva com a tabela de recorrências.
4
Cota Inferior Ω(n) do Problema
Pense: "qual é o mínimo de operações que QUALQUER algoritmo precisa fazer?". Geralmente é Ω(n) (precisa ler a entrada toda) ou Ω(log n) (busca binária). Justifique.
5
Comparação e Conclusão de Otimalidade
Compare T(n) com Ω(n). Se atingem a mesma classe, escreva: "O algoritmo é Θ(n) e a cota inferior do problema é Ω(n). Portanto, o algoritmo é ÓTIMO." Caso contrário, é honesto dizer: "Não é ótimo, pois T(n) = O(2ⁿ) >> Ω(n)."

Erros Comuns que Descontam Nota
✗ Esquecer de definir T(n)
O Strauss literalmente escreve "Lembre-se de obrigatoriamente definir a função tempo!" em quase toda questão. Não definir T(n) é perder pontos automaticamente.
✗ Confundir complexidade do algoritmo com Ω do problema
"Meu algoritmo é Θ(n log n), portanto o problema é Ω(n log n)" — isto é um erro grave. A cota inferior é uma propriedade do problema, não do seu algoritmo. Para Fibonacci, o algoritmo é O(2ⁿ) mas Ω do problema é apenas Ω(n).
✗ Não justificar a escolha da estrutura de dados
"Usarei uma lista de adjacência" não basta. Sempre adicione: "...porque o grafo é esparso e preciso iterar nos vizinhos em O(grau(u))."
✗ Algoritmo iterativo onde se pede recursivo
As questões com "Desenvolva UM ÚNICO algoritmo RECURSIVO" exigem solução recursiva. Iterativa = nota zero na questão.
✗ Pseudocódigo vago ou sem caso base
Toda recursão precisa de caso base explícito. Faltando = recursão infinita = nota zero. Sempre escreva o if caso_base: return ... primeiro.
✗ Não testar manualmente o algoritmo
Quando a questão pede "aplique o algoritmo para [exemplo concreto]", você precisa realmente executar passo a passo, não apenas escrever o algoritmo. Mostre as iterações.
