Por que analisar complexidade?
Antes de qualquer algoritmo, precisamos responder: quanto tempo e memória ele consome à medida que a entrada cresce? Um algoritmo que demora 1ms para n=100 pode demorar 11 dias para n=1.000.000 se for O(n²) — ou ainda 1 segundo se for O(n log n). As notações assintóticas são a linguagem matemática que nos permite comparar algoritmos independentemente do hardware, da linguagem e das constantes de implementação.
Ω
Ômega — Limite Inferior
O algoritmo precisa de pelo menos esse tempo. É o melhor caso assintótico — cota de baixo.
f(n) = Ω(g(n))
∃ c,n₀: f(n) ≥ c·g(n), n≥n₀
O
Big-O — Limite Superior
O algoritmo demora no máximo esse tempo. É o pior caso assintótico — cota de cima.
f(n) = O(g(n))
∃ c,n₀: f(n) ≤ c·g(n), n≥n₀
Θ
Theta — Limite Exato
O algoritmo demora exatamente essa ordem. Limita superior e inferiormente — crescimento "sanduichado".
f(n) = Θ(g(n))
f = O(g) e f = Ω(g)
Definição formal e intuição
NotaçãoLeitura informalExemplo concretoUso
Ω(n) "cresce pelo menos tão rápido quanto n" Ordenação por comparação: Ω(n log n) Provar que não há algoritmo mais rápido
O(n²) "cresce no máximo tão rápido quanto n²" Bubble sort: O(n²) no pior caso Garantir que o algoritmo caberá no tempo
Θ(n log n) "cresce exatamente nessa ordem" Merge sort: Θ(n log n) em todos os casos Caracterização completa do algoritmo
📐 Regra de ouro: Quando alguém diz "o algoritmo é O(n²)", normalmente querem dizer que o pior caso cresce quadraticamente. Big-O é a notação mais usada na prática por ser a mais conservadora — garante que o programa nunca ultrapassa aquele tempo.

Regras de Simplificação
# Regra 1: Constantes são descartadas
f(n) = 3n² + 5n + 100  →  O(n²)  # termo dominante

# Regra 2: Multiplicação de loops aninhados
for i in range(n):
    for j in range(n):  # → O(n²)
        fazer_algo()

# Regra 3: Soma de etapas sequenciais → maior domina
busca_linear(arr)   # O(n)
ordenar(arr)        # O(n log n)
# Total: O(n + n log n) = O(n log n)

# Regra 4: Ω(limite inferior) para ordenação por comparação
# Qualquer algoritmo de ordenação por comparação: Ω(n log n)
# Merge sort é Θ(n log n) → é ótimo nessa classe!

# Regra 5: log n em qualquer base → mesma classe
O(log₂ n) = O(log₁₀ n) = O(log n)  # diferença é só constante
⚠ Armadilha comum: O(n²) não significa "lento sempre". Para n=10, O(n²)=100 operações e O(n log n)≈33. A diferença é irrelevante. Complexidade importa para grandes entradas. Sempre pergunte: qual é o n do meu problema?

O que são algoritmos de ordenação?

🎯 Por que ordenar importa?

Imagine uma pilha de 1.000.000 de registros. Desordenados: busca em O(n) = 1.000.000 operações. Ordenados: busca binária em O(log n) = apenas 20 operações. Ordenar é uma das operações mais fundamentais da computação — e não existe "o melhor algoritmo": existe o certo para cada situação.
