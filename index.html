Análise de Eficiência e Complexidade Linear O(n)

1. O Problema da Celebridade

Lógica O(n): Utilizamos uma abordagem de eliminação. Assumimos que a primeira pessoa é a candidata. Iteramos pela multidão: se o candidato atual conhece a próxima pessoa, ele não pode ser a celebridade, então a próxima pessoa se torna o novo candidato. Ao final de uma passagem, temos um único candidato válido que deve ser verificado em uma segunda passagem. 
def encontrar_celebridade(matriz_conhecimento, n):
    candidato = 0
    
    # Fase 1: Eliminação para achar o candidato potencial em O(n)
    for pessoa in range(1, n):
        if matriz_conhecimento[candidato][pessoa] == True:
            # O candidato conhece alguém, logo não é celebridade.
            candidato = pessoa

    # Fase 2: Verificar se o candidato restante é realmente a celebridade em O(n)
    for pessoa in range(n):
        if pessoa != candidato:
            # Regra: Candidato não pode conhecer ninguém e todos devem conhecê-lo
            if matriz_conhecimento[candidato][pessoa] or not matriz_conhecimento[pessoa][candidato]:
                return -1 # Não há celebridade
                
    return candidato

2. Eleição por Maioria Absoluta

Lógica O(n): Para encontrar o elemento majoritário usando tempo linear e espaço constante, usamos o Algoritmo de Votação de Boyer-Moore. Mantemos um contador: se o voto for igual ao do candidato atual, incrementamos; se for diferente, decrementamos. Se o contador zerar, o voto atual vira o novo candidato. 
def vencedor_maioria_absoluta(votos):
    candidato_atual = None
    saldo_votos = 0

    # Fase 1: Descobrir o candidato mais provável (Boyer-Moore) em O(n)
    for voto in votos:
        if saldo_votos == 0:
            candidato_atual = voto
            saldo_votos = 1
        elif voto == candidato_atual:
            saldo_votos += 1
        else:
            saldo_votos -= 1

    # Fase 2: Validar se o candidato realmente tem mais de 50% em O(n)
    votos_totais = len(votos)
    contagem_real = sum(1 for v in votos if v == candidato_atual)

    if contagem_real > votos_totais // 2:
        return candidato_atual
    return None # Ninguém obteve maioria absoluta

3. Busca na Agenda Telefônica

Lógica O(n) vs O(1): Em estruturas de dados modernas (como Dicionários/Hash Tables), a busca é $O(1)$. No entanto, se a agenda for uma lista não ordenada, precisamos de uma busca linear O(n). Abaixo, a representação linear. 
def buscar_amigo_agenda(agenda_lista, nome_buscado):
    # Busca Linear O(n) iterando sobre a lista de contatos
    for contato in agenda_lista:
        if contato['nome'] == nome_buscado:
            return contato['telefone']
            
    return "Contato não encontrado"
    
# Nota teórica: Se usássemos Python dict (Hash Table):
# telefone = agenda_dicionario.get(nome_buscado) -> O(1)

4. Troca de Mensagem Segura e Anônima

Lógica: Este é um problema de Criptografia Assimétrica (Chaves Públicas e Privadas). Para garantir confidencialidade, o remente criptografa a mensagem com a Chave Pública do receptor. Para garantir autoria, ele assina com sua própria Chave Privada. O processamento caractere por caractere (ou bloco a bloco) garante uma complexidade de tempo linear O(n) em relação ao tamanho da mensagem. 
def preparar_mensagem_segura(mensagem, chave_privada_remetente, chave_publica_receptor):
    mensagem_processada = []
    
    # A complexidade é O(n) onde 'n' é o tamanho da mensagem
    for caractere in mensagem:
        # 1. Assinatura: Criptografa com a chave privada de quem envia (Garante autoria)
        char_assinado = assinar_caractere(caractere, chave_privada_remetente)
        
        # 2. Confidencialidade: Criptografa com a chave pública do destino (Só ele lê)
        char_seguro = criptografar_caractere(char_assinado, chave_publica_receptor)
        
        mensagem_processada.append(char_seguro)
        
    return mensagem_processada

5. Otimização da Distribuição de Jornais

Lógica O(n): Como o número de pontos de distribuição é fixo ($K = 3$), iteramos sobre as $n$ bancas e calculamos a distância para os 3 pontos. A complexidade fica $O(K \times n)$, que como $K$ é constante, simplifica-se para um problema perfeitamente linear $O(n)$. 
import math

def calcular_distancia(coord1, coord2):
    # Teorema de Pitágoras simples - Operação O(1)
    return math.sqrt((coord1[0] - coord2[0])**2 + (coord1[1] - coord2[1])**2)

def otimizar_rotas_jornais(bancas, pontos_distribuicao):
    # Inicializa a alocação de rotas
    alocacao = {ponto: [] for ponto in pontos_distribuicao}
    
    # Para cada banca (n), comparamos com os 3 pontos fixos: Tempo O(n)
    for banca in bancas:
        menor_distancia = float('inf')
        melhor_ponto = None
        
        for ponto in pontos_distribuicao:
            distancia = calcular_distancia(banca, ponto)
            
            if distancia < 

 menor_distancia:
                menor_distancia = distancia
                melhor_ponto = ponto
                
        alocacao[melhor_ponto].append(banca)
        
    return alocacao
