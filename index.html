Por que analisar complexidade?
Antes de qualquer algoritmo, precisamos responder: quanto tempo e memória ele consome à medida que a entrada cresce? Um algoritmo que demora 1ms para n=100 pode demorar 11 dias para n=1.000.000 se for O(n²) — ou ainda 1 segundo se for O(n log n). As notações assintóticas são a linguagem matemática que nos permite comparar algoritmos independentemente do hardware, da linguagem e das constantes de implementação.
Ω
Ômega — Limite Inferior
O algoritmo precisa de pelo menos esse tempo. É o melhor caso assintótico — cota de baixo.
f(n) = Ω(g(n))
∃ c,n₀: f(n) ≥ c·g(n), n≥n₀
O
Big-O — Limite Superior
O algoritmo demora no máximo esse tempo. É o pior caso assintótico — cota de cima.
f(n) = O(g(n))
∃ c,n₀: f(n) ≤ c·g(n), n≥n₀
Θ
Theta — Limite Exato
O algoritmo demora exatamente essa ordem. Limita superior e inferiormente — crescimento "sanduichado".
f(n) = Θ(g(n))
f = O(g) e f = Ω(g)

Árvores Binárias

Tema: Estrutura Cai em ~75% das P1's Recursão essencial
Por que estudar isso primeiro: entre as 4 últimas provas P1 do Prof. Strauss, 3 cobraram reconstrução de árvore binária a partir de percursos. Dominar este tópico significa garantir uma questão inteira da prova.
Definições Fundamentais
Uma árvore binária é uma estrutura recursiva onde cada nó tem no máximo dois filhos: o filho esquerdo e o filho direito. Formalmente, uma árvore binária é:
  • vazia (representada por null ou nil), ou
  • composta por um nó raiz + uma subárvore esquerda + uma subárvore direita, ambas binárias.
  • 01Raiz: nó no topo da árvore, sem pai.
  • 02Folha: nó sem filhos (subárvores esquerda e direita vazias).
  • 03Altura: número de arestas no caminho mais longo da raiz até uma folha. Árvore vazia tem altura -1; árvore com só raiz tem altura 0.
  • 04Profundidade de um nó: número de arestas da raiz até ele.
  • 05Grau: número de filhos do nó (0, 1 ou 2 em árvores binárias).
  • 06Árvore cheia: todo nó interno tem exatamente 2 filhos. Completa: todos os níveis preenchidos exceto possivelmente o último, que é preenchido da esquerda para a direita.
Os Quatro Percursos
Um percurso (traversal) é uma forma sistemática de visitar todos os nós de uma árvore. Como árvores são estruturas recursivas, todos os percursos são definidos recursivamente. A diferença está em quando visitamos a raiz:

# Implementação dos 4 percursos (recursivos, exceto em-nível)
class Node:
    def __init__(self, valor):
        self.valor = valor
        self.esq = None
        self.dir = None

def preordem(no):
    if no is None: return
    print(no.valor)         # 1. visita a raiz
    preordem(no.esq)        # 2. percorre subárvore esquerda
    preordem(no.dir)        # 3. percorre subárvore direita

def inordem(no):           # também chamada de SIMÉTRICA
    if no is None: return
    inordem(no.esq)
    print(no.valor)
    inordem(no.dir)

def posordem(no):
    if no is None: return
    posordem(no.esq)
    posordem(no.dir)
    print(no.valor)

from collections import deque
def em_nivel(raiz):       # BFS — usa fila
    if raiz is None: return
    fila = deque([raiz])
    while fila:
        no = fila.popleft()
        print(no.valor)
        if no.esq: fila.append(no.esq)
        if no.dir: fila.append(no.dir)
Análise de Complexidade dos Percursos
Função tempo T(n) dos três percursos recursivos. Considere uma árvore com n nós: 
T(0) = c1                          # caso base (no == None)
T(n) = T(esq) + T(dir) + c2          # processa raiz + 2 chamadas recursivas

Cada nó é visitado exatamente UMA vez → operações O(1) por nó
T(n) = n · O(1) = Θ(n)
Conclusão: os percursos são ótimos — não dá para visitar n nós em menos que Θ(n) operações, que é o limite inferior Ω(n) do problema.

Algoritmo de Reconstrução (Pré-ordem + In-ordem)
  • 01O primeiro elemento da pré-ordem é a raiz da árvore.
  • 02Localize essa raiz no vetor in-ordem (posição k).
  • 03Tudo à esquerda de k no in-ordem é a subárvore esquerda. Tudo à direita, a subárvore direita.
  • 04Os próximos k elementos da pré-ordem (após a raiz) formam a pré-ordem da subárvore esquerda. Os restantes formam a pré-ordem da direita.
  • 05Recursão: aplique os passos 1-4 nas duas metades.
def reconstroi(preordem, inordem):
    if not preordem or not inordem:
        return None

    # 1. Primeiro da pré-ordem é a raiz
    raiz_valor = preordem[0]
    raiz = Node(raiz_valor)

    # 2. Localiza a raiz no in-ordem
    k = inordem.index(raiz_valor)   # O(n) na implementação ingênua

    # 3. Divide os percursos
    inord_esq = inordem[:k]
    inord_dir = inordem[k+1:]
    pre_esq   = preordem[1:k+1]
    pre_dir   = preordem[k+1:]

    # 4. Recursão
    raiz.esq = reconstroi(pre_esq, inord_esq)
    raiz.dir = reconstroi(pre_dir, inord_dir)
    return raiz
Análise da Função Tempo
Implementação ingênua (acima): 
T(n) = T(k) + T(n-k-1) + O(n)    # O(n) é o custo de inordem.index()
                                  
Pior caso (árvore degenerada k=0 ou k=n-1):
T(n) = T(n-1) + O(n)  →  T(n) = O(n²)

Caso médio (k ≈ n/2):
T(n) = 2·T(n/2) + O(n)  →  T(n) = O(n log n)
Otimização para Θ(n): usar HashMap para mapear valor → índice no in-ordem em pré-processamento O(n). Aí a busca vira O(1) e: 
T(n) = T(k) + T(n-k-1) + O(1)  →  T(n) = Θ(n)
Como Ω(n) é o mínimo (precisamos olhar todos os nós), o algoritmo otimizado é ÓTIMO.

Notações Matemáticas: Infixa, Prefixa, Pósfixa

Cai em ~50% das P1's Árvore + Recursão Conversão
Conexão direta com 1.3: toda expressão matemática pode ser representada como uma árvore binária. Os operadores são nós internos, os operandos são folhas. A escolha da notação é literalmente escolher um percurso dessa árvore.
As Três Notações
NotaçãoPosição do operadorEquivale a percursoExemplo (3+4)·5
InfixaEntre os operandosIn-ordem (com parênteses)(3 + 4) * 5
Prefixa (Polonesa)Antes dos operandosPré-ordem* + 3 4 5
Pósfixa (Polonesa Reversa)Depois dos operandosPós-ordem3 4 + 5 *
Por que isto importa?
  • Infixa é a notação humana, mas precisa de parênteses e regras de precedência.
  • Prefixa e pósfixa não precisam de parênteses nem precedência — são totalmente determinadas pela posição. Por isso são usadas internamente em compiladores e calculadoras.
  • A pósfixa é avaliada em tempo linear com uma pilha: cada operando é empilhado, cada operador desempilha 2 valores, calcula, empilha o resultado.

Algoritmo Recursivo: Prefixa → Infixa + Pósfixa
Questão clássica do Prof. Strauss: "Desenvolva UM ÚNICO algoritmo recursivo que transforme uma função matemática escrita na forma PREFIXA para as formas INFIXA e PÓSFIXA. Descreva se sua proposta é ótima." 
def converte_prefixa(tokens, idx=0):
    """
    Lê tokens prefixa e retorna (infixa, posfixa, novo_idx).
    Estratégia: um único algoritmo que constrói as duas formas
    simultaneamente em uma única passada recursiva.
    """
    if idx >= len(tokens):
        return "", "", idx

    token = tokens[idx]

    if eh_operador(token):
        # 1. processa subárvore esquerda
        inf_e, pos_e, idx = converte_prefixa(tokens, idx + 1)
        # 2. processa subárvore direita
        inf_d, pos_d, idx = converte_prefixa(tokens, idx)
        # 3. monta as duas formas a partir das subexpressões
        infixa  = f"({inf_e} {token} {inf_d})"
        posfixa = f"{pos_e} {pos_d} {token}"
        return infixa, posfixa, idx

    else:  # é operando (folha)
        return token, token, idx + 1

# Uso:
# tokens = ["+", "Z", "*", "Q", "/", "T", "L"]
# infixa, posfixa, _ = converte_prefixa(tokens)
Análise da Função Tempo
Cada token é visitado exatamente uma vez e processado em O(1): 
T(0) = c1                              # vetor vazio
T(n) = T(nesq) + T(ndir) + c2            # operador: 2 chamadas recursivas
T(n) = c3                              # operando: caso base

Como nesq + ndir + 1 = n (cada nó conta uma vez):
T(n) = Θ(n)

Cota inferior: Ω(n)  →  precisamos ler todos os tokens
Algoritmo: Θ(n)      →  ATINGE A COTA  →  É ÓTIMO ✓

Algoritmos Recursivos

Tema obrigatório Pilha de chamadas Recorrências
Recursão é a linguagem natural de árvores e divide-and-conquer. O Prof. Strauss exige soluções recursivas em quase toda questão. Esta seção te ensina a analisar a função tempo de algoritmos recursivos — o ponto central de toda prova.
Anatomia de um Algoritmo Recursivo
Todo algoritmo recursivo bem escrito tem três componentes:
  • Caso base: condição em que a função retorna sem chamar a si mesma. Sem caso base → recursão infinita → stack overflow.
  • Caso recursivo: a função chama a si mesma com um problema menor que se aproxima do caso base.
  • Combinação: como o resultado das chamadas recursivas é combinado para resolver o problema original.
# Esqueleto genérico
def resolve(problema):
    if eh_caso_base(problema):
        return solucao_trivial(problema)        # caso base

    # Decompõe em subproblemas menores
    sub1, sub2 = decompoe(problema)

    # Resolve recursivamente
    res1 = resolve(sub1)
    res2 = resolve(sub2)

    # Combina os resultados
    return combina(res1, res2)
Estudo de Caso: Fibonacci
Sequência de Fibonacci: fib(0)=0, fib(1)=1, fib(n) = fib(n-1) + fib(n-2). Apareceu na P1-2018.01 (5ª questão), pedindo para analisar a árvore de chamadas e determinar Ω e O. 
def fib(n):
    if n <= 1:                 # caso base: O(1)
        return n
    return fib(n - 1) + fib(n - 2)  # 2 chamadas recursivas

Análise: Por que é O(2ⁿ)?
A função tempo do Fibonacci recursivo ingênuo é: 
T(0) = T(1) = c1                  # casos base
T(n) = T(n-1) + T(n-2) + c2        # caso recursivo

Esta é a própria recorrência de Fibonacci! Pode-se mostrar que:
T(n) = Θ(φⁿ) onde φ = (1+√5)/2 ≈ 1.618 (razão áurea)

Cota superior: T(n) ≤ 2·T(n-1) + c  →  T(n) = O(2ⁿ)
Cota inferior: T(n) ≥ 2·T(n-2) + c  →  T(n) = Ω((√2)ⁿ)

Número exato de chamadas: 2·fib(n+1) - 1
Para n=4: 2·fib(5) - 1 = 2·5 - 1 = 9 chamadas (a prova diz 8, contando diferente)
⚠ É ÓTIMO? NÃO. A cota inferior do problema "calcular fib(n)" é Ω(n) (precisamos no mínimo executar n adições). Como o algoritmo ingênuo é O(2ⁿ) >> Ω(n), ele NÃO é ótimo. A versão iterativa ou com memoização atinge Θ(n) e é ótima.

Como Escrever T(n) na Prova (Passo a Passo)
  • 01Identifique o caso base: "para n = 0 (ou 1, conforme), T(n) = c1 (constante)".
  • 02Conte as chamadas recursivas: quantas vezes a função se chama? Em quais tamanhos?
  • 03Mensure o trabalho não-recursivo: tudo que NÃO é chamada recursiva — comparações, cópias, prints. Pode ser O(1), O(n), O(log n)...
  • 04Escreva a recorrência: T(n) = (soma das T's das chamadas) + (trabalho extra).
  • 05Resolva: use a tabela de recorrências comuns acima ou expanda manualmente.
  • 06Compare com a cota inferior Ω(n) do PROBLEMA: se T(n) atinge Ω(n), o algoritmo é ótimo.
Exemplo completo de resposta (Fibonacci recursivo): 
1. Caso base:    T(0) = T(1) = c1
2. Caso recurs.: 2 chamadas (n-1 e n-2), trabalho extra: 1 soma = c2
3. Recorrência:  T(n) = T(n-1) + T(n-2) + c2
4. Solução:      T(n) = Θ(φⁿ) ≈ Θ(1.618ⁿ)  →  O(2ⁿ)
5. Cota inferior do problema "calcular fib(n)": Ω(n)
6. Comparação: O(2ⁿ) >> Ω(n)  →  NÃO é ótimo

Questão clássica do Prof. Strauss: "Desenvolva um algoritmo que, dado um texto, faça a identificação do total de palíndromos existentes de palavra e de frases, separadamente." (P1-2023.02 — Q2)
Definição
Um palíndromo é uma sequência de caracteres que se lê igual da esquerda para a direita e da direita para a esquerda. Existem dois níveis de palíndromo importantes na prova:
  • Palíndromo de palavra: a palavra inteira é palíndromo. Ex: HANNAH, OSSO, RADAR.
  • Palíndromo de frase: a frase, ignorando espaços, pontuação e capitalização, é palíndromo. Ex: ROMA ME TEM AMOR → ROMAMETEMAMOR é palíndromo.
⚠ Pegadinha: em frases, é necessário pré-processar removendo espaços, acentos e pontuação. Sem isso, "ROMA ME TEM AMOR" tem 17 caracteres, mas "ROMA" começa com R e "AMOR" termina com R — só funciona se você ignorar os espaços.
Algoritmo de Dois Ponteiros
  • 01Pré-processe a string: converta para maiúsculas, remova espaços e pontuação.
  • 02Crie dois ponteiros: i no início (0) e j no final (n-1).
  • 03Compare s[i] com s[j]. Se diferentes, não é palíndromo.
  • 04Se iguais, avance: i++ e j--.
  • 05Repita até i ≥ j. Se chegou aqui, é palíndromo.
def eh_palindromo(s):
    # 1. Pré-processa
    s = ''.join(c.upper() for c in s if c.isalnum())

    # 2. Dois ponteiros
    i, j = 0, len(s) - 1
    while i < j:
        if s[i] != s[j]:
            return False
        i += 1
        j -= 1
    return True

# Identificação de palíndromos no texto:
def conta_palindromos(texto):
    palavras = texto.split()
    # palavra-palíndromo: cada palavra individual
    pal_palavras = sum(1 for w in palavras if eh_palindromo(w) and len(w) > 1)
    # frase-palíndromo: testa cada frase (separadas por . ! ?)
    import re
    frases = re.split(r'[.!?]', texto)
    pal_frases = sum(1 for f in frases if f.strip() and eh_palindromo(f))
    return pal_palavras, pal_frases

Algoritmo de Euclides — MDC
O algoritmo de Euclides calcula o Máximo Divisor Comum (MDC, ou GCD em inglês) entre dois inteiros. É um dos algoritmos mais antigos da matemática, descrito por Euclides em ~300 a.C., e ainda assim é o mais eficiente conhecido. 
Identidade fundamental: mdc(a, b) = mdc(b, a mod b), com mdc(a, 0) = a
def mdc(a, b):
    # Recursivo (mais elegante)
    if b == 0:
        return a
    return mdc(b, a % b)

def mdc_iterativo(a, b):
    # Iterativo (mais rápido na prática)
    while b != 0:
        a, b = b, a % b
    return a

def eh_primo(n):
    # Caso base
    if n < 2: return False
    if n == 2: return True
    if n % 2 == 0: return False

    # Testa só ímpares de 3 até √n
    i = 3
    while i * i <= n:
        if n % i == 0: return False
        i += 2
    return True

def primos_vizinhos(n):
    """Encontra os 2 primos antes e os 2 depois de n"""
    antecessores = []
    p = n - 1
    while len(antecessores) < 2 and p >= 2:
        if eh_primo(p): antecessores.append(p)
        p -= 1
    sucessores = []
    p = n + 1
    while len(sucessores) < 2:
        if eh_primo(p): sucessores.append(p)
        p += 1
    return antecessores, sucessores

Fundamentos: Distância e Cobertura
Distância entre dois pontos (Pitágoras): 
d((x1,y1), (x2,y2)) = √((x2-x1)² + (y2-y1)²)
Truque importante: para verificar se um ponto P está dentro de um círculo C de centro (xc, yc) e raio R, NÃO é necessário calcular a raiz quadrada. Basta comparar quadrados: 
(x - xc)² + (y - yc)² ≤ R²    →    P está DENTRO do círculo
Isso evita uma operação cara (sqrt) e o resultado é exato com inteiros. 
def ponto_dentro_circulo(px, py, cx, cy, raio):
    # SEM raiz — compara quadrados
    dx = px - cx
    dy = py - cy
    return dx*dx + dy*dy <= raio*raio

def quadro_seguro(quadro, cameras):
    """
    Retorna True se o quadro está coberto por AO MENOS uma câmera.
    quadro: (px, py)
    cameras: lista de (cx, cy, raio)
    """
    px, py = quadro
    for cx, cy, raio in cameras:
        if ponto_dentro_circulo(px, py, cx, cy, raio):
            return True   # basta UMA câmera cobrir
    return False

Grafos — Teoria e Algoritmos

Aplicação: Rede e Museu Pt.2 BFS · DFS · Dijkstra Estrutura universal

Definições Fundamentais
Um grafo G = (V, E) é uma estrutura formada por:
  • V: conjunto de vértices (nós).
  • E: conjunto de arestas, onde cada aresta conecta um par de vértices.
Tipos de grafos:
  • Direcionado: arestas têm direção (u → v ≠ v → u).
  • Não-direcionado: arestas são bidirecionais.
  • Ponderado: cada aresta tem um peso (ex: distância, custo).
  • Conexo: existe caminho entre todo par de vértices.
  • 01Grau de um vértice: número de arestas incidentes a ele. Em grafos direcionados, há grau de entrada (in-degree) e grau de saída (out-degree).
  • 02Caminho: sequência de vértices onde cada par consecutivo é conectado por aresta. Comprimento = nº de arestas.
  • 03Ciclo: caminho fechado (vértice inicial = final).
  • 04Árvore: grafo conexo, não-direcionado, sem ciclos. Tem exatamente V−1 arestas.
Representações em Memória
Existem duas representações principais. A escolha depende do problema:

Matriz de Adjacência

M[u][v] = 1 se aresta uv existe
# Vetor 2D V × V
adj = [[0] * V for _ in range(V)]
adj[u][v] = 1
adj[v][u] = 1  # não-direcionado

Espaço: Θ(V²) sempre.
Adjacência? O(1).
Vizinhos de u? O(V).
Use quando: grafos densos, ou quando V é pequeno.

Lista de Adjacência

adj[u] = [vizinhos de u]
# Vetor de listas
adj = [[] for _ in range(V)]
adj[u].append(v)
adj[v].append(u)  # não-direcionado

Espaço: Θ(V + E).
Adjacência? O(grau(u)).
Vizinhos de u? O(grau(u)).
Use quando: grafos esparsos (E << V²), ou para BFS/DFS.

Busca em Largura (BFS)
Visita os vértices em ordem de distância a partir da origem. Encontra o caminho com menor número de arestas em grafos não-ponderados. Usa fila (FIFO). 
from collections import deque

def bfs(adj, origem):
    visitados = {origem}
    fila = deque([origem])
    ordem = []
    while fila:
        u = fila.popleft()
        ordem.append(u)
        for v in adj[u]:
            if v not in visitados:
                visitados.add(v)
                fila.append(v)
    return ordem

# Complexidade: cada vértice é visitado 1 vez (V) e cada aresta também (E)
# T(V, E) = Θ(V + E)
Busca em Profundidade (DFS)
Vai o mais fundo possível antes de retroceder. Usa pilha (LIFO) — implementação recursiva é a mais natural. Útil para detectar ciclos, ordenação topológica, componentes conexos. 
def dfs(adj, u, visitados=None, ordem=None):
    if visitados is None: visitados = set()
    if ordem is None: ordem = []
    visitados.add(u)
    ordem.append(u)
    for v in adj[u]:
        if v not in visitados:
            dfs(adj, v, visitados, ordem)
    return ordem

# Complexidade: Θ(V + E) — mesma análise da BFS

Caminho Mínimo (Dijkstra) — Aplicação no Museu
Para a Parte 2 do problema do Museu (trajetória do ladrão evitando câmeras), modelamos a sala como um grafo: pontos seguros (não cobertos por câmera) são vértices conectados por arestas com peso = distância. Dijkstra encontra o caminho mais curto. 
import heapq

def dijkstra(adj, origem, n):
    """
    adj: dict {u: [(v, peso), ...]}
    Retorna lista dist[v] = menor distância de origem a v.
    """
    dist = [float('inf')] * n
    dist[origem] = 0
    heap = [(0, origem)]   # (distância, vértice)

    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]: continue   # versão obsoleta
        for v, peso in adj[u]:
            nd = d + peso
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(heap, (nd, v))
    return dist

# Complexidade: O((V + E) log V) com heap binário

O Problema da Celebridade

Tempo: O(n) Espaço: O(1) Eliminação por candidato
Antes: §1.3 Estruturas de Dados Pilha
Enunciado: Em uma multidão de n pessoas, todos conhecem uma celebridade, mas a celebridade não conhece ninguém. Dado um oráculo conhece(A, B) que responde em O(1), encontre a celebridade com o menor número possível de chamadas.
Intuição
A chave do problema está em explorar a assimetria do relacionamento. Se perguntarmos "A conhece B?" e a resposta for sim — A imediatamente se descarta como celebridade, pois celebridade não conhece ninguém. Se a resposta for não — B se descarta, pois todos conhecem a celebridade.

Com isso, cada pergunta elimina exatamente um candidato. Com n-1 perguntas, ficamos com um único candidato sobrevivente. Uma segunda rodada de validação em O(n) confirma se ele é de fato a celebridade. Total: O(n).
Passo a Passo
  • 01Inicialize o candidato = 0 (qualquer pessoa serve como ponto de partida).
  • 02Para cada pessoa i de 1 até n-1: se candidato conhece i, então candidato = i (o antigo foi eliminado). Caso contrário, i é eliminado (candidato sobrevive).
  • 03Ao fim da Fase 1, temos um único sobrevivente. Ele pode ser a celebridade, mas precisamos verificar.
  • 04Fase 2: verifique (a) que o candidato não conhece ninguém e (b) que todos conhecem o candidato. Se ambas as condições falham, não há celebridade.
Implementação
def encontrar_celebridade(conhece, n):
    # ─── Fase 1: Eliminação ───────────────────────────────────────────
    # Invariante: candidato nunca foi eliminado
    candidato = 0

    for i in range(1, n):
        if conhece(candidato, i):
            # candidato conhece i → candidato NÃO pode ser celebridade
            candidato = i
        # else: i não é conhecido pelo candidato → i eliminado

    # ─── Fase 2: Validação ────────────────────────────────────────────
    for pessoa in range(n):
        if pessoa == candidato:
            continue
        # A celebridade não conhece ninguém E todos conhecem ela
        if conhece(candidato, pessoa) or not conhece(pessoa, candidato):
            return -1  # Não existe celebridade

    return candidato

⚠ A abordagem ingênua — checar todos os pares — faz O(n²) consultas. O algoritmo acima é ótimo: qualquer algoritmo correto precisa de pelo menos Ω(n) consultas, pois precisa ver cada pessoa ao menos uma vez.

Votação por Maioria Absoluta

Tempo: O(n) Espaço: O(1) Algoritmo de Boyer-Moore
Antes: §1.3 Estruturas de Dados Contagem
Enunciado: Dado o histórico de votos de uma eleição, determine se existe um candidato com maioria absoluta (mais de 50% dos votos + 1). Se existir, retorne-o. Use espaço constante.
Intuição
O algoritmo de Boyer-Moore é uma obra de arte da elegância algorítmica. Imagine que cada voto é uma moeda de um candidato. Quando dois candidatos diferentes se encontram, eles se "cancelam" mutuamente — removemos um de cada. O elemento que sobrar no fim é o único que poderia ter maioria, pois se um elemento aparece mais de n/2 vezes, ele sempre sobrevive ao cancelamento.

Porém, sobreviver não garante maioria — é necessária uma segunda passagem para confirmar a contagem real.
Passo a Passo
  • 01Inicialize candidato = None e contagem = 0.
  • 02Para cada voto: se contagem é 0, o candidato passa a ser o voto atual. Se o voto é igual ao candidato, contagem += 1. Caso contrário, contagem -= 1 (cancelamento).
  • 03Fase 2: conte quantas vezes o candidato sobrevivente aparece. Se mais de n/2 vezes, ele tem maioria absoluta.
Implementação
def maioria_absoluta(votos):
    # ─── Fase 1: Encontrar candidato por Boyer-Moore ──────────────────
    candidato = None
    contagem  = 0

    for voto in votos:
        if contagem == 0:
            candidato = voto       # Novo candidato emerge
            contagem  = 1
        elif voto == candidato:
            contagem += 1         # Reforço: mesmo candidato
        else:
            contagem -= 1         # Cancelamento: votos diferentes se anulam

    # ─── Fase 2: Confirmar se tem maioria real ────────────────────────
    confirmacao = sum(1 for v in votos if v == candidato)

    if confirmacao > len(votos) / 2:
        return candidato
    return None  # Nenhum candidato com maioria absoluta

votos = ['X', 'Y', 'X', 'X', 'Y', 'X', 'X']
print(maioria_absoluta(votos))  # → 'X' (5 de 7 votos)
💡 Por que não usar um dicionário de contagens? Funcionaria em O(n) tempo, mas usaria O(k) espaço onde k é o número de candidatos. Boyer-Moore faz o mesmo em O(1) espaço — ideal para streams de dados gigantescas onde não cabe tudo na memória.

Agenda de Contatos do Celular

Busca: O(log n) Espaço: O(n) Lista ordenada + Busca binária
Antes: §1.3 Estruturas de Dados Busca Binária
Enunciado: Como o celular armazena e encontra um contato na agenda de forma eficiente? Modele a estrutura de dados ideal considerando as operações: adicionar contato, remover contato, buscar por nome e listar em ordem alfabética.
Intuição
Uma lista desordenada exigiria varrer todos os contatos a cada busca — O(n) por pesquisa. Mas a grande maioria das agendas mantém os contatos ordenados alfabeticamente. Com isso, podemos usar Busca Binária: a cada passo, descartamos metade dos contatos restantes. Com 1.000 contatos, precisamos de no máximo 10 comparações (log₂ 1000 ≈ 10).
Análise das Operações

Implementação
import bisect

class AgendaContatos:
    def __init__(self):
        self.contatos = []    # Lista de dicts ordenada por 'nome'
        self.nomes    = []    # Lista paralela só com nomes (para bisect)

    def adicionar(self, nome, telefone):
        pos = bisect.bisect_left(self.nomes, nome)
        self.nomes.insert(pos, nome)
        self.contatos.insert(pos, {'nome': nome, 'tel': telefone})

    def buscar(self, nome):
        pos = bisect.bisect_left(self.nomes, nome)
        if pos < len(self.nomes) and self.nomes[pos] == nome:
            return self.contatos[pos]['tel']
        return "Não encontrado"

    def remover(self, nome):
        pos = bisect.bisect_left(self.nomes, nome)
        if pos < len(self.nomes) and self.nomes[pos] == nome:
            self.nomes.pop(pos)
            self.contatos.pop(pos)

Criptografia com Anonimato de Remetente

Tempo: O(n) Espaço: O(1) extra Tabela Hash + Cifra de Substituição
Antes: §1.9 Teoria dos Números Mat. Modular
Enunciado: Envie uma mensagem de modo que somente o receptor final saiba o conteúdo e ninguém saiba quem enviou. Modele o mecanismo de criptografia da mensagem e anonimização do remetente.
Intuição
O problema tem duas dimensões independentes: (1) criptografar o conteúdo e (2) ocultar a identidade do remetente. Para o conteúdo, usamos uma Tabela Hash como chave de substituição — cada caractere é mapeado para outro em O(1). Para o anonimato, modelamos um sistema de roteamento em cadeia (onion routing simplificado): a mensagem passa por intermediários que não sabem nem a origem nem o destino final.
Implementação
import hashlib, os

# ─── Parte 1: Cifra de Substituição via Tabela Hash ───────────────
def gerar_chave_substituicao(seed=42):
    import random
    rng = random.Random(seed)
    alfabeto = list("ABCDEFGHIJKLMNOPQRSTUVWXYZ")
    embaralhado = alfabeto[:]
    rng.shuffle(embaralhado)
    chave   = dict(zip(alfabeto, embaralhado))
    inversa = dict(zip(embaralhado, alfabeto))
    return chave, inversa

def criptografar(mensagem, chave):
    return "".join(chave.get(c.upper(), c) for c in mensagem)

def descriptografar(cifrado, inversa):
    return "".join(inversa.get(c, c) for c in cifrado)

chave, inversa = gerar_chave_substituicao()
msg_original   = "ENCONTRO NA SEXTA"
msg_cifrada    = criptografar(msg_original, chave)
print(msg_cifrada)
print(descriptografar(msg_cifrada, inversa))  # → "ENCONTRO NA SEXTA"
🔐 Em criptografia real moderna, usa-se AES (simétrico) ou RSA/Elliptic Curve (assimétrico). O modelo de roteamento anônimo é a base do Tor (The Onion Router). A cifra de substituição é vulnerável à análise de frequência, mas ilustra perfeitamente o uso de hash maps em O(1).

Distribuição de Jornais com 3 Centros

Tempo: O(n) Espaço: O(n) K-means com K fixo (K=3)
Antes: §1.10 Geometria Clustering
Enunciado: Um bairro tem n bancas de jornal e 3 pontos de distribuição fixos em posições aleatórias. Otimize a distribuição alocando cada banca ao ponto mais próximo, minimizando a distância total percorrida.
Intuição
Como K=3 é uma constante fixa, o loop interno que compara cada banca com os 3 centros executa exatamente 3 vezes — um número constante, independente de n. Logo, O(K×n) = O(3×n) = O(n). Esse é um padrão importante: quando K é fixo, o que parece quadrático colapsa para linear.
Implementação
import math

def distancia_euclidiana(ponto_a, ponto_b):
    return math.sqrt((ponto_a[0] - ponto_b[0])**2 + (ponto_a[1] - ponto_b[1])**2)

def alocar_bancas(bancas, centros):
    alocacao = {c: [] for c in centros}
    for banca in bancas:               # Loop externo: O(n)
        melhor_centro = min(           # Loop interno: O(K) = O(3) = O(1)
            centros,
            key=lambda c: distancia_euclidiana(banca, c)
        )
        alocacao[melhor_centro].append(banca)
    return alocacao

centros = [(2, 3), (8, 7), (5, 1)]
bancas  = [(1, 2), (9, 6), (4, 1), (7, 8), (3, 4)]
resultado = alocar_bancas(bancas, centros)

Fila de Prioridade com Prioridades Dinâmicas

Extração: O(log n) Espaço: O(n) Heap Binário (Min-Heap)
Antes: §1.3 Estruturas de Dados Heap
Enunciado: Um sistema de tarefas tem prioridades que mudam dinamicamente durante a execução. Como garantir que sempre executemos a tarefa de maior prioridade, mesmo com inserções e mudanças de prioridade frequentes?
Intuição
Uma lista simples exigiria O(n) para encontrar a tarefa de maior prioridade a cada extração. Um Heap Binário resolve isso em O(log n): é uma árvore binária quase-completa onde o pai sempre tem prioridade maior que os filhos. O elemento de maior prioridade está sempre na raiz — acesso em O(1). Inserção e remoção reorganizam a árvore em no máximo log₂n passos.
Passo a Passo
  • 01Inserção: Adiciona ao fim do heap. Então "sobe" (heapify-up): compara com o pai e troca se necessário. Máximo de log n trocas.
  • 02Extração do máximo: Remove a raiz. Move o último elemento para a raiz. "Desce" (heapify-down): compara com filhos e troca com o maior. Máximo de log n trocas.
  • 03Atualização de prioridade: Localiza a tarefa (O(1) com dicionário auxiliar). Atualiza a prioridade. Se aumentou, sobe; se diminuiu, desce. O(log n).
Implementação
import heapq

class FilaPrioridadeDinamica:
    def __init__(self):
        self.heap   = []
        self.index  = {}
        self.contador = 0
        self.REMOVIDA = 'REMOVIDA'

    def inserir(self, tarefa, prioridade):
        if tarefa in self.index:
            self.atualizar(tarefa, prioridade)
            return
        entrada = [-prioridade, self.contador, tarefa]
        self.index[tarefa] = entrada
        heapq.heappush(self.heap, entrada)
        self.contador += 1

    def atualizar(self, tarefa, nova_prioridade):
        entrada_antiga = self.index.pop(tarefa)
        entrada_antiga[2] = self.REMOVIDA
        self.inserir(tarefa, nova_prioridade)

    def extrair_maior(self):
        while self.heap:
            prioridade, _, tarefa = heapq.heappop(self.heap)
            if tarefa != self.REMOVIDA:
                del self.index[tarefa]
                return tarefa, -prioridade

fila = FilaPrioridadeDinamica()
fila.inserir("Relatório", 3)
fila.inserir("Email urgente", 8)
fila.atualizar("Relatório", 10)  # Prazo se aproximou!
print(fila.extrair_maior())   # → ("Relatório", 10)

Top-5 Funcionários por Produtividade

Tempo: O(n log 5) = O(n) Espaço: O(1) Min-Heap de Tamanho Fixo
Antes: §1.3 Estruturas de Dados Heap
Enunciado: Uma empresa quer selecionar os 5 funcionários mais produtivos de um total de n para premiação em um resort. Como fazer isso sem ordenar todos os funcionários?
Intuição
Ordenar todos os n funcionários custaria O(n log n). Mas precisamos apenas dos 5 melhores. A solução elegante é manter um min-heap de tamanho 5. A raiz guarda o menor dos top-5. Para cada novo funcionário, se ele for melhor que o pior do top-5, substituímos. Assim o heap sempre contém os 5 melhores vistos até agora, e cada atualização custa O(log 5) = O(1).
Implementação
import heapq

def top_k_funcionarios(funcionarios, k=5):
    heap = []

    for nome, prod in funcionarios:        # O(n)
        if len(heap) < k:
            heapq.heappush(heap, (prod, nome))
        elif prod > heap[0][0]:
            heapq.heapreplace(heap, (prod, nome))  # O(log k)

    return sorted(heap, key=lambda x: -x[0])

# Complexidade Final: n × O(log k) = O(n log k)
# k=5 constante → O(n log 5) = O(n). Muito melhor que O(n log n)!

funcionarios = [
    ("Alice", 92), ("Bruno", 87), ("Carla", 95),
    ("Diego", 78), ("Eva", 99), ("Fábio", 88),
]
premiados = top_k_funcionarios(funcionarios)
📊 Esse padrão (top-K com K fixo usando min-heap) é amplamente usado em sistemas de recomendação, rankings em tempo real e processamento de streams de dados onde não podemos armazenar tudo na memória.

Linha de Montagem com Máquinas Heterogêneas

Tempo: O(n × m) Espaço: O(n) Fila de Prioridade + Escalonamento Greedy
Antes: §1.3 Estruturas de Dados Heap · Greedy
Enunciado: Em situação emergencial, maximize a produção de uma fábrica com múltiplas linhas de montagem (m linhas), cada uma com máquinas modernas (rápidas) e antigas (lentas). Como escalonar a produção para maximizar o throughput?
Intuição
Este é um problema de escalonamento de tarefas em máquinas paralelas heterogêneas. A estratégia gulosa ótima é: sempre alocar o próximo item à máquina que terminar primeiro. Isso evita que máquinas rápidas fiquem ociosas enquanto lentas acumulam trabalho.

Usamos um min-heap de disponibilidade: a raiz sempre aponta para a máquina que ficará livre primeiro.
Implementação
import heapq

def escalonar_producao(n_pecas, linhas_maquinas):
    heap = []
    for linha_id, maquinas in enumerate(linhas_maquinas):
        for maq_id, velocidade in enumerate(maquinas):
            heapq.heappush(heap, (0, linha_id, maq_id, velocidade))

    tempo_final = 0
    for peca in range(n_pecas):
        tempo_livre, linha, maquina, vel = heapq.heappop(heap)
        novo_tempo = tempo_livre + vel
        tempo_final = max(tempo_final, novo_tempo)
        heapq.heappush(heap, (novo_tempo, linha, maquina, vel))

    return tempo_final

# 2 linhas: máquina moderna (vel=2) e antiga (vel=5) em cada
linhas = [[2, 5], [3, 6]]
print(escalonar_producao(8, linhas))

Operações com Matrizes M×N

Soma: O(M·N) Mult: O(M·N·P) Álgebra Linear + Arrays 2D
Antes: §1.1 Notações & Complexidade Arrays 2D
Enunciado: Implemente operações de soma e multiplicação de matrizes com análise de complexidade. Para a soma: A[M×N] + B[M×N] = C[M×N]. Para a multiplicação: A[M×N] × B[N×P] = C[M×P].
Implementação
def soma_matrizes(A, B):
    M, N = len(A), len(A[0])
    return [[A[i][j] + B[i][j] for j in range(N)] for i in range(M)]  # O(M·N)

def mult_matrizes(A, B):
    M = len(A); N = len(A[0]); P = len(B[0])
    C = [[0]*P for _ in range(M)]
    for i in range(M):       # O(M)
        for j in range(P):   # O(P)
            for k in range(N): # O(N)
                C[i][j] += A[i][k] * B[k][j]
    return C  # Complexidade total: O(M·N·P)

Torre de Hanói — Recursivo vs Iterativo

Tempo: O(2ⁿ) Espaço Rec: O(n) Recursão + Pilha explícita
Antes: §1.2 Recursão Recursão
Enunciado: Mova n discos do pino A para o pino C usando o pino B como auxiliar. Nunca coloque um disco maior sobre um menor. Implemente e compare a versão recursiva com a iterativa (usando pilha explícita).
Intuição
A recorrência é T(n) = 2T(n-1) + 1. Resolvendo: T(n) = 2ⁿ - 1. Para n=10: 1.023 movimentos. Para n=64: mais de 18 quintilhões — levaria bilhões de anos. A versão iterativa usa uma pilha explícita para simular o call stack da recursão, com o mesmo número de movimentos mas sem overhead de chamadas de função.
Implementação
def hanoi_rec(n, origem, destino, auxiliar, movimentos=[]):
    if n == 1:
        movimentos.append((origem, destino))
        return
    hanoi_rec(n-1, origem, auxiliar, destino, movimentos)   # Move n-1 para aux
    movimentos.append((origem, destino))                        # Move o maior
    hanoi_rec(n-1, auxiliar, destino, origem, movimentos)   # Move n-1 de aux → dest
    return movimentos

# T(n) = 2T(n-1) + 1  →  T(n) = 2ⁿ - 1 movimentos
# Espaço: O(n) na call stack (profundidade da recursão)
def hanoi_iter(n, origem, destino, auxiliar):
    # Pilha explícita simula o call stack da recursão
    pilha = [(n, origem, destino, auxiliar)]
    movimentos = []

    while pilha:
        n, o, d, a = pilha.pop()
        if n == 1:
            movimentos.append((o, d))
            continue
        # Empilha em ordem inversa (pilha é LIFO)
        pilha.append((n-1, a, d, o))    # 3: move n-1 aux→dest
        pilha.append((1, o, d, a))       # 2: move disco maior
        pilha.append((n-1, o, a, d))    # 1: move n-1 orig→aux

    return movimentos  # Mesmos 2ⁿ-1 movimentos, sem recursão!
// C++ — versão recursiva clássica
