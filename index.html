1. O Problema da Celebridade

Tempo: O(n) Espaço: O(1)
Lógica: Utilizamos uma abordagem de eliminação. Assumimos que a primeira pessoa é a candidata. Iteramos pela multidão: se o candidato atual conhece a próxima pessoa, ele é imediatamente eliminado. A segunda passagem apenas valida as conexões do único candidato restante, garantindo a complexidade de tempo linear e espaço constante. 
def encontrar_celebridade(matriz_conhecimento, n):
    candidato = 0
    
    # Fase 1: Eliminação em O(n)
    for pessoa in range(1, n):
        if matriz_conhecimento[candidato][pessoa]: # Se o candidato conhece a pessoa
            candidato = pessoa

    # Fase 2: Validação em O(n)
    for pessoa in range(n):
        if pessoa != candidato:
            # Regra: Candidato não conhece ninguém E todos conhecem o candidato
            if matriz_conhecimento[candidato][pessoa] or not matriz_conhecimento[pessoa][candidato]:
                return -1
                
    return candidato

Grafo de Conhecimento: A Celebridade (amarela) recebe arestas de todos, mas não emite nenhuma.

2. Eleição por Maioria Absoluta

Tempo: O(n) Espaço: O(1)
Lógica: Aplicamos o Algoritmo de Votação de Boyer-Moore. Mantemos um contador: se o voto for igual ao do candidato atual, incrementamos; se diferente, decrementamos. Pares de votos divergentes se anulam. O candidato que "sobra" no saldo é o provável vencedor, confirmado em uma segunda varredura. 
def vencedor_maioria(votos):
    candidato = None
    saldo = 0

    # Fase 1: Encontrar o candidato provável em O(n)
    for v in votos:
        if saldo == 0:
            candidato = v
            saldo = 1
        elif v == candidato:
            saldo += 1
        else:
            saldo -= 1

    # Fase 2: Confirmar se possui mais de 50% dos votos em O(n)
    if votos.count(candidato) > len(votos) // 2:
        return candidato
    return None

Visualização de Votos: Elementos de mesma cor reforçam o saldo; cores diferentes se anulam até sobrar a maioria.

3. Agenda Telefônica Dinâmica

Busca/Remoção: O(n) Adição: O(1) Ordenação: O(n log n)
Lógica de Operações: Em uma lista sequencial, a inserção ao final tem custo O(1) amortizado. A busca e a remoção exigem percorrer os elementos, resultando em tempo linear O(n). Para a ordenação, a linguagem Python utiliza o Timsort, um algoritmo híbrido altamente otimizado que opera em O(n log n). 
def adicionar_contato(agenda, nome, telefone):
    # Adição no fim da lista: O(1) amortizado
    agenda.append({'nome': nome, 'telefone': telefone})

def remover_contato(agenda, nome):
    # Compreensão de lista para filtrar e remover o alvo: O(n)
    agenda[:] = [contato for contato in agenda if contato['nome'] != nome]

def ordenar_agenda(agenda):
    # Ordenação In-place (Timsort interno do Python): O(n log n)
    agenda.sort(key=lambda x: x['nome'])

def buscar_contato(agenda, nome):
    # Busca Linear: O(n)
    for contato in agenda:
        if contato['nome'] == nome:
            return contato['telefone']
    return "Contato não encontrado"

Estrutura Sequencial: Representação dos contatos como nós que podem ser iterados, adicionados ou removidos.

4. Criptografia por Tabela Hash

Tempo: O(n) Espaço: O(1) extra
Lógica: Utilizamos um Dicionário Python (Tabela Hash) para implementar uma Cifra de Substituição. O custo para buscar a correspondência de um caractere na tabela hash é estritamente constante, O(1). Portanto, processar e criptografar uma mensagem de tamanho n resulta em uma complexidade total O(n). O espaço extra é O(1) pois o tamanho do alfabeto mapeado é constante. 
def gerar_tabela_hash_cripto():
    # Tabela Hash O(1) atuando como chave de substituição
    return {'A': 'X', 'B': 'Y', 'C': 'Z', 'D': 'W', 'E': 'V'}

def criptografar_mensagem(mensagem, tabela_hash):
    msg_criptografada = []
    
    for letra in mensagem:
        # O método .get() busca na Tabela Hash em tempo O(1)
        letra_cifrada = tabela_hash.get(letra, letra)
        msg_criptografada.append(letra_cifrada)
        
    # O join agrupa a lista final em O(n)
    return "".join(msg_criptografada)

Mapeamento Hash: Chaves (Texto Claro) apontam diretamente para Valores (Texto Cifrado) em tempo constante.

5. Otimização de Distribuição de Jornais

Tempo: O(n) Espaço: O(n)
Lógica: Sendo K = 3 o número de pontos de distribuição fixos, iteramos pelas n bancas calculando a distância euclidiana para os 3 centros. Como K é uma constante limitante, a complexidade O(K * n) simplifica-se perfeitamente para um tempo linear O(n). 
import math

def otimizar_rotas_jornais(bancas, pontos_distribuicao):
    # Inicializa o dicionário de rotas
    alocacao = {ponto: [] for ponto in pontos_distribuicao}
    
    # Loop O(n) percorrendo todas as bancas
    for banca in bancas:
        menor_distancia = float('inf')
        melhor_ponto = None
        
        # Loop O(1) restrito aos 3 pontos constantes
        for ponto in pontos_distribuicao:
            dist = math.sqrt((banca[0] - ponto[0])**2 + (banca[1] - ponto[1])**2)
            
            if dist < menor_distancia:
                menor_distancia = dist
                melhor_ponto = ponto
                
        alocacao[melhor_ponto].append(banca)
        
    return alocacao

Alocação Espacial: Bancas (pontos azuis) ligam-se ao Centro de Distribuição (estrelas vermelhas) mais próximo.
