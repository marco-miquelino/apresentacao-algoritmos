1. O Problema da Celebridade

Lógica O(n): Assumimos que a primeira pessoa é a candidata. Iteramos pela multidão: se o candidato atual conhece a próxima pessoa, ele é eliminado. A segunda passagem apenas valida as conexões do único candidato restante.

Grafo de Conhecimento: A Celebridade (amarela) recebe arestas de todos, mas não emite nenhuma.

2. Eleição por Maioria Absoluta

Lógica O(n): Pelo Algoritmo de Boyer-Moore, pares de votos divergentes se anulam. O candidato que "sobra" no saldo é o provável vencedor, confirmado em uma segunda passagem. 
def vencedor_maioria(votos):
    candidato, saldo = None, 0
    for v in votos:
        if saldo == 0: candidato, saldo = v, 1
        elif v == candidato: saldo += 1
        else: saldo -= 1
    return candidato if votos.count(candidato) > len(votos)//2 else None

Visualização de Votos: Elementos de mesma cor reforçam o saldo, cores diferentes se anulam até sobrar a maioria.

3. Agenda Telefônica Dinâmica

Lógica de Operações: Adição no fim da lista é O(1). A remoção e a busca linear são O(n). A ordenação utiliza o algoritmo nativo do Python (Timsort), que opera em O(n log n). 
def adicionar_contato(agenda, nome, tel):
    agenda.append({'nome': nome, 'tel': tel}) # O(1) amortizado

def remover_contato(agenda, nome):
    # Filtra a lista removendo o alvo em O(n)
    agenda[:] = [c for c in agenda if c['nome'] != nome]

def ordenar_agenda(agenda):
    # Ordenação In-place em O(n log n)
    agenda.sort(key=lambda x: x['nome'])

def buscar_contato(agenda, nome):
    for c in agenda:
        if c['nome'] == nome: return c['tel'] # O(n)

Estrutura Sequencial: Representação dos contatos como nós que podem ser reorganizados, adicionados ou removidos.

Lógica O(1) por caractere: Utilizamos um Dicionário (Tabela Hash) para criar uma Cifra de Substituição. O custo para encontrar a letra correspondente na tabela é O(1). Logo, percorrer uma mensagem de tamanho n custa O(n) no total.

def criptografar_mensagem(mensagem, tabela_hash):
    msg_criptografada = []
    for letra in mensagem:
        # Busca na Hash Table: O(1)
        letra_cifrada = tabela_hash.get(letra, letra)
        msg_criptografada.append(letra_cifrada)
        
    return "".join(msg_criptografada) # Complexidade Final: O(n)

Mapeamento Hash: Chaves (Texto Claro) apontam diretamente para Valores (Texto Cifrado) em tempo constante.

5. Otimização de Jornais

Lógica O(n): Com K=3 pontos fixos de distribuição, calculamos a distância de cada uma das n bancas para os 3 centros. A operação O(3*n) simplifica-se para O(n).

Alocação Espacial: Bancas (pontos azuis) ligam-se ao Centro de Distribuição (estrelas vermelhas) mais próximo.
