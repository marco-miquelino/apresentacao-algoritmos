01

O Problema da Celebridade

Tempo: O(n) Espaço: O(1) Eliminação por candidato
Enunciado: Em uma multidão de n pessoas, todos conhecem uma celebridade, mas a celebridade não conhece ninguém. Dado um oráculo conhece(A, B) que responde em O(1), encontre a celebridade com o menor número possível de chamadas.
Intuição
A chave do problema está em explorar a assimetria do relacionamento. Se perguntarmos "A conhece B?" e a resposta for sim — A imediatamente se descarta como celebridade, pois celebridade não conhece ninguém. Se a resposta for não — B se descarta, pois todos conhecem a celebridade.

Com isso, cada pergunta elimina exatamente um candidato. Com n-1 perguntas, ficamos com um único candidato sobrevivente. Uma segunda rodada de validação em O(n) confirma se ele é de fato a celebridade. Total: O(n).
Passo a Passo
  • 01Inicialize o candidato = 0 (qualquer pessoa serve como ponto de partida).
  • 02Para cada pessoa i de 1 até n-1: se candidato conhece i, então candidato = i (o antigo foi eliminado). Caso contrário, i é eliminado (candidato sobrevive).
  • 03Ao fim da Fase 1, temos um único sobrevivente. Ele pode ser a celebridade, mas precisamos verificar.
  • 04Fase 2: verifique (a) que o candidato não conhece ninguém e (b) que todos conhecem o candidato. Se ambas as condições falham, não há celebridade.
Implementação
def encontrar_celebridade(conhece, n):
    # ─── Fase 1: Eliminação ───────────────────────────────────────────
    # Invariante: candidato nunca foi eliminado
    candidato = 0

    for i in range(1, n):
        if conhece(candidato, i):
            # candidato conhece i → candidato NÃO pode ser celebridade
            candidato = i
        # else: i não é conhecido pelo candidato → i eliminado

    # ─── Fase 2: Validação ────────────────────────────────────────────
    for pessoa in range(n):
        if pessoa == candidato:
            continue
        # A celebridade não conhece ninguém E todos conhecem ela
        if conhece(candidato, pessoa) or not conhece(pessoa, candidato):
            return -1  # Não existe celebridade

    return candidato
FaseOperaçõesComplexidade
Eliminaçãon-1 chamadas a conhece()O(n)
Validaçãoaté 2(n-1) chamadasO(n)
Totalmáximo 3(n-1) chamadasO(n)
⚠ A abordagem ingênua — checar todos os pares — faz O(n²) consultas. O algoritmo acima é ótimo: qualquer algoritmo correto precisa de pelo menos Ω(n) consultas, pois precisa ver cada pessoa ao menos uma vez.
Visualização

Todos apontam para a celebridade (dourado). Nenhuma aresta sai dela.

02

Votação por Maioria Absoluta

Tempo: O(n) Espaço: O(1) Algoritmo de Boyer-Moore
Enunciado: Dado o histórico de votos de uma eleição, determine se existe um candidato com maioria absoluta (mais de 50% dos votos + 1). Se existir, retorne-o. Use espaço constante.
Intuição
O algoritmo de Boyer-Moore é uma obra de arte da elegância algorítmica. Imagine que cada voto é uma moeda de um candidato. Quando dois candidatos diferentes se encontram, eles se "cancelam" mutuamente — removemos um de cada. O elemento que sobrar no fim é o único que poderia ter maioria, pois se um elemento aparece mais de n/2 vezes, ele sempre sobrevive ao cancelamento.

Porém, sobreviver não garante maioria — é necessária uma segunda passagem para confirmar a contagem real.
Passo a Passo
  • 01Inicialize candidato = None e contagem = 0.
  • 02Para cada voto: se contagem é 0, o candidato passa a ser o voto atual. Se o voto é igual ao candidato, contagem += 1. Caso contrário, contagem -= 1 (cancelamento).
  • 03Fase 2: conte quantas vezes o candidato sobrevivente aparece. Se mais de n/2 vezes, ele tem maioria absoluta.
Implementação
def maioria_absoluta(votos):
    # ─── Fase 1: Encontrar candidato por Boyer-Moore ──────────────────
    candidato = None
    contagem  = 0

    for voto in votos:
        if contagem == 0:
            candidato = voto       # Novo candidato emerge
            contagem  = 1
        elif voto == candidato:
            contagem += 1         # Reforço: mesmo candidato
        else:
            contagem -= 1         # Cancelamento: votos diferentes se anulam

    # ─── Fase 2: Confirmar se tem maioria real ────────────────────────
    # O sobrevivente pode não ter maioria (ex: [A, B, C])
    confirmacao = sum(1 for v in votos if v == candidato)

    if confirmacao > len(votos) / 2:
        return candidato
    return None  # Nenhum candidato com maioria absoluta


# Exemplo:
votos = ['X', 'Y', 'X', 'X', 'Y', 'X', 'X']
print(maioria_absoluta(votos))  # → 'X' (5 de 7 votos)
💡 Por que não usar um dicionário de contagens? Funcionaria em O(n) tempo, mas usaria O(k) espaço onde k é o número de candidatos. Boyer-Moore faz o mesmo em O(1) espaço — ideal para streams de dados gigantescas onde não cabe tudo na memória.
Visualização

Votos do mesmo candidato somam; votos diferentes se cancelam mutuamente.

Agenda de Contatos do Celular

Busca: O(log n) Espaço: O(n) Lista ordenada + Busca binária
Enunciado: Como o celular armazena e encontra um contato na agenda de forma eficiente? Modele a estrutura de dados ideal considerando as operações: adicionar contato, remover contato, buscar por nome e listar em ordem alfabética.
Intuição
Uma lista desordenada exigiria varrer todos os contatos a cada busca — O(n) por pesquisa. Mas a grande maioria das agendas mantém os contatos ordenados alfabeticamente. Com isso, podemos usar Busca Binária: a cada passo, descartamos metade dos contatos restantes. Com 1.000 contatos, precisamos de no máximo 10 comparações (log₂ 1000 ≈ 10).

Na prática, celulares modernos usam árvores de busca balanceadas (como B-trees) ou índices hash, mas a busca binária em lista ordenada é o modelo conceitual correto.
Análise das Operações
OperaçãoEstruturaComplexidade
Buscar por nomeLista ordenada + Busca BináriaO(log n)
Adicionar contatoInserção em posição corretaO(n) (shift)
Remover contatoDeleção com shiftO(n)
Listar ordenadoLista já mantida ordenadaO(n)
Implementação
import bisect

class AgendaContatos:
    def __init__(self):
        self.contatos = []    # Lista de dicts ordenada por 'nome'
        self.nomes    = []    # Lista paralela só com nomes (para bisect)

    def adicionar(self, nome, telefone):
        # bisect_left devolve o índice de inserção para manter a ordem: O(log n)
        pos = bisect.bisect_left(self.nomes, nome)
        self.nomes.insert(pos, nome)                           # O(n) shift
        self.contatos.insert(pos, {'nome': nome, 'tel': telefone})

    def buscar(self, nome):
        # Busca Binária: O(log n) comparações
        pos = bisect.bisect_left(self.nomes, nome)
        if pos < len(self.nomes) and self.nomes[pos] == nome:
            return self.contatos[pos]['tel']
        return "Não encontrado"

    def remover(self, nome):
        # Localiza com busca binária, depois remove: O(log n) + O(n) shift
        pos = bisect.bisect_left(self.nomes, nome)
        if pos < len(self.nomes) and self.nomes[pos] == nome:
            self.nomes.pop(pos)
            self.contatos.pop(pos)

    def listar(self):
        # Já ordenado! O(n) para imprimir
        return self.contatos
Visualização

Lista ordenada alfabeticamente. A busca binária elimina metade dos contatos a cada comparação.

Criptografia com Anonimato de Remetente

Tempo: O(n) Espaço: O(1) extra Tabela Hash + Cifra de Substituição
Enunciado: Envie uma mensagem de modo que somente o receptor final saiba o conteúdo e ninguém saiba quem enviou. Modele o mecanismo de criptografia da mensagem e anonimização do remetente.
Intuição
O problema tem duas dimensões independentes: (1) criptografar o conteúdo e (2) ocultar a identidade do remetente. Para o conteúdo, usamos uma Tabela Hash como chave de substituição — cada caractere é mapeado para outro em O(1). Para o anonimato, modelamos um sistema de roteamento em cadeia (onion routing simplificado): a mensagem passa por intermediários que não sabem nem a origem nem o destino final.
Implementação
import hashlib, os

# ─── Parte 1: Cifra de Substituição via Tabela Hash ───────────────
def gerar_chave_substituicao(seed=42):
    """Gera um mapeamento bijetivo de cada letra para outra."""
    import random
    rng = random.Random(seed)
    alfabeto = list("ABCDEFGHIJKLMNOPQRSTUVWXYZ")
    embaralhado = alfabeto[:]
    rng.shuffle(embaralhado)
    # Chave (dict): busca em O(1) por hash interno do Python
    chave    = dict(zip(alfabeto, embaralhado))
    inversa  = dict(zip(embaralhado, alfabeto))
    return chave, inversa

def criptografar(mensagem, chave):
    # Cada lookup na tabela hash é O(1) → total O(n)
    return "".join(chave.get(c.upper(), c) for c in mensagem)

def descriptografar(cifrado, inversa):
    return "".join(inversa.get(c, c) for c in cifrado)

# ─── Parte 2: Anonimato via Cadeia de Roteamento ──────────────────
class NoRoteamento:
    def __init__(self, id_no):
        self.id = id_no

    def repassar(self, pacote, proximo_no):
        # O nó intermediário vê apenas o próximo salto, nunca a origem
        print(f"Nó {self.id}: repassando para {proximo_no.id} (origem desconhecida)")
        return proximo_no

# Uso combinado:
chave, inversa = gerar_chave_substituicao()
msg_original   = "ENCONTRO NA SEXTA"
msg_cifrada    = criptografar(msg_original, chave)
print(msg_cifrada)       # Ex: "VQPBZQCF QU YVMZU"
print(descriptografar(msg_cifrada, inversa))  # → "ENCONTRO NA SEXTA"
🔐 Em criptografia real moderna, usa-se AES (simétrico) ou RSA/Elliptic Curve (assimétrico). O modelo de roteamento anônimo é a base do Tor (The Onion Router). A cifra de substituição é vulnerável à análise de frequência, mas ilustra perfeitamente o uso de hash maps em O(1).
Visualização

Tabela Hash mapeia cada caractere em tempo O(1) — independente do tamanho da mensagem.

Distribuição de Jornais com 3 Centros

Tempo: O(n) Espaço: O(n) K-means com K fixo (K=3)
Enunciado: Um bairro tem n bancas de jornal e 3 pontos de distribuição fixos em posições aleatórias. Otimize a distribuição alocando cada banca ao ponto mais próximo, minimizando a distância total percorrida.
Intuição
Como K=3 é uma constante fixa, o loop interno que compara cada banca com os 3 centros executa exatamente 3 vezes — um número constante, independente de n. Logo, O(K×n) = O(3×n) = O(n). Esse é um padrão importante: quando K é fixo, o que parece quadrático colapsa para linear.

Este problema é uma versão simplificada do K-means clustering, mas com centros fixos (não há iteração de convergência). O critério de alocação é a distância euclidiana mínima.
Implementação
import math

def distancia_euclidiana(ponto_a, ponto_b):
    # Teorema de Pitágoras em 2D
    return math.sqrt((ponto_a[0] - ponto_b[0])**2 + (ponto_a[1] - ponto_b[1])**2)

def alocar_bancas(bancas, centros):
    """
    bancas  : lista de tuplas (x, y) — posição de cada banca
    centros : lista de 3 tuplas (x, y) — pontos de distribuição fixos
    
    Retorna: dicionário centro → [lista de bancas atribuídas]
    """
    alocacao = {c: [] for c in centros}

    for banca in bancas:               # Loop externo: O(n)
        melhor_centro = min(           # Loop interno: O(K) = O(3) = O(1)
            centros,
            key=lambda c: distancia_euclidiana(banca, c)
        )
        alocacao[melhor_centro].append(banca)

    return alocacao

def distancia_total(alocacao):
    """Calcula a soma de todas as distâncias (métrica de qualidade)."""
    total = 0
    for centro, bancas_grupo in alocacao.items():
        for banca in bancas_grupo:
            total += distancia_euclidiana(banca, centro)
    return total

# Exemplo de uso:
centros = [(2, 3), (8, 7), (5, 1)]
bancas  = [(1, 2), (9, 6), (4, 1), (7, 8), (3, 4)]
resultado = alocar_bancas(bancas, centros)
print(f"Distância total: {distancia_total(resultado):.2f}")
Visualização

Cada banca (azul) conecta ao centro de distribuição (vermelho) mais próximo.

Fila de Prioridade com Prioridades Dinâmicas

Extração: O(log n) Espaço: O(n) Heap Binário (Min-Heap)
Enunciado: Um sistema de tarefas tem prioridades que mudam dinamicamente durante a execução. Como garantir que sempre executemos a tarefa de maior prioridade, mesmo com inserções e mudanças de prioridade frequentes?
Intuição
Uma lista simples exigiria O(n) para encontrar a tarefa de maior prioridade a cada extração. Um Heap Binário resolve isso em O(log n): é uma árvore binária quase-completa onde o pai sempre tem prioridade maior que os filhos. O elemento de maior prioridade está sempre na raiz — acesso em O(1). Inserção e remoção reorganizam a árvore em no máximo log₂n passos.

Para prioridades dinâmicas, mantemos um mapeamento de tarefa→índice no heap, permitindo localizar e atualizar a prioridade de qualquer tarefa em O(log n).
Passo a Passo
  • 01Inserção: Adiciona ao fim do heap. Então "sobe" (heapify-up): compara com o pai e troca se necessário. Máximo de log n trocas.
  • 02Extração do máximo: Remove a raiz. Move o último elemento para a raiz. "Desce" (heapify-down): compara com filhos e troca com o maior. Máximo de log n trocas.
  • 03Atualização de prioridade: Localiza a tarefa (O(1) com dicionário auxiliar). Atualiza a prioridade. Se aumentou, sobe; se diminuiu, desce. O(log n).
Implementação
import heapq

class FilaPrioridadeDinamica:
    def __init__(self):
        self.heap   = []   # Min-heap: (−prioridade, contador, tarefa)
        self.index  = {}   # tarefa → entrada ativa no heap
        self.contador = 0  # Desempate por ordem de chegada (FIFO)
        self.REMOVIDA = 'REMOVIDA'  # Marcador de entrada invalidada

    def inserir(self, tarefa, prioridade):
        if tarefa in self.index:
            self.atualizar(tarefa, prioridade)
            return
        entrada = [-prioridade, self.contador, tarefa]  # negativo → max-heap via min-heap
        self.index[tarefa] = entrada
        heapq.heappush(self.heap, entrada)              # O(log n)
        self.contador += 1

    def atualizar(self, tarefa, nova_prioridade):
        # Invalidação lazy: marca a entrada antiga como removida
        entrada_antiga = self.index.pop(tarefa)
        entrada_antiga[2] = self.REMOVIDA
        self.inserir(tarefa, nova_prioridade)  # Insere nova entrada

    def extrair_maior(self):
        while self.heap:
            prioridade, _, tarefa = heapq.heappop(self.heap)  # O(log n)
            if tarefa != self.REMOVIDA:
                del self.index[tarefa]
                return tarefa, -prioridade
        raise KeyError("Fila vazia")

# Exemplo:
fila = FilaPrioridadeDinamica()
fila.inserir("Relatório", 3)
fila.inserir("Email urgente", 8)
fila.inserir("Reunião", 5)
fila.atualizar("Relatório", 10)  # Prazo aproximou — prioridade sobe!
print(fila.extrair_maior())   # → ("Relatório", 10)
Visualização

Heap Binário: o elemento de maior prioridade sempre está na raiz.

Top-5 Funcionários por Produtividade

Tempo: O(n log 5) = O(n) Espaço: O(1) Min-Heap de Tamanho Fixo
Enunciado: Uma empresa quer selecionar os 5 funcionários mais produtivos de um total de n para premiação em um resort. Como fazer isso sem ordenar todos os funcionários?
Intuição
Ordenar todos os n funcionários custaria O(n log n). Mas precisamos apenas dos 5 melhores. A solução elegante é manter um min-heap de tamanho 5. A raiz guarda o menor dos top-5. Para cada novo funcionário, se ele for melhor que o pior do top-5, substituímos. Assim o heap sempre contém os 5 melhores vistos até agora, e cada atualização custa O(log 5) = O(1).
Implementação
import heapq

def top_k_funcionarios(funcionarios, k=5):
    """
    funcionarios: lista de (nome, produtividade)
    k           : número de funcionários premiados
    
    Estratégia: min-heap de tamanho k
    - Raiz = o MENOR dos top-k (o que seria eliminado primeiro)
    - Se novo funcionário > raiz → substitui e reheapifica
    """
    heap = []  # Min-heap com no máximo k elementos

    for nome, prod in funcionarios:        # O(n)
        if len(heap) < k:
            heapq.heappush(heap, (prod, nome))  # Preenche as k primeiras vagas
        elif prod > heap[0][0]:              # Candidato melhor que o pior do top-k
            heapq.heapreplace(heap, (prod, nome))  # Remove o pior, insere novo: O(log k)

    # Extrai do maior para o menor
    return sorted(heap, key=lambda x: -x[0])

# Complexidade Final:
# n iterações × O(log k) por heapreplace = O(n log k)
# Como k=5 é constante → O(n log 5) = O(n)
# Muito melhor que O(n log n) de uma ordenação completa!

funcionarios = [
    ("Alice", 92), ("Bruno", 87), ("Carla", 95),
    ("Diego", 78), ("Eva", 99), ("Fábio", 88),
]
premiados = top_k_funcionarios(funcionarios, k=5)
for prod, nome in premiados:
    print(f"{nome}: {prod} pontos")
📊 Esse padrão (top-K com K fixo usando min-heap) é amplamente usado em sistemas de recomendação, rankings em tempo real e processamento de streams de dados onde não podemos armazenar tudo na memória.
Visualização

Min-heap de tamanho 5: a raiz é o menor dos top-5. Novos candidatos desafiam a raiz.

08

Linha de Montagem com Máquinas Heterogêneas

Tempo: O(n × m) Espaço: O(n) Fila de Prioridade + Escalonamento Greedy
Enunciado: Em situação emergencial, maximize a produção de uma fábrica com múltiplas linhas de montagem (m linhas), cada uma com máquinas modernas (rápidas) e antigas (lentas). Como escalonar a produção para maximizar o throughput?
Intuição
Este é um problema de escalonamento de tarefas em máquinas paralelas heterogêneas. A estratégia gulosa ótima é: sempre alocar o próximo item à máquina que terminar primeiro. Isso evita que máquinas rápidas fiquem ociosas enquanto lentas acumulam trabalho.

Usamos um min-heap de disponibilidade: a raiz sempre aponta para a máquina que ficará livre primeiro. Quando uma nova peça chega, extraímos a máquina mais disponível, alocamos, e reinserimos com o novo tempo de conclusão.
Implementação
import heapq

def escalonar_producao(n_pecas, linhas_maquinas):
    """
    n_pecas        : número de peças a produzir
    linhas_maquinas: lista de listas — cada sub-lista tem os tempos
                     de produção das máquinas naquela linha
                     Ex: [[2, 5], [3, 6]] = 2 linhas, máquinas com
                     velocidades 2 e 5 (moderna e antiga, resp.)
    
    Retorna: tempo total até última peça ser concluída (makespan)
    """
    # Inicializa o heap com todas as máquinas disponíveis no tempo 0
    # Formato: (tempo_livre, linha_id, maquina_id, velocidade)
    heap = []
    for linha_id, maquinas in enumerate(linhas_maquinas):
        for maq_id, velocidade in enumerate(maquinas):
            heapq.heappush(heap, (0, linha_id, maq_id, velocidade))

    log        = []
    tempo_final = 0

    for peca in range(n_pecas):
        # Máquina que ficará livre mais cedo: O(log(total_maquinas))
        tempo_livre, linha, maquina, vel = heapq.heappop(heap)

        novo_tempo = tempo_livre + vel  # Tempo de conclusão desta peça
        tempo_final = max(tempo_final, novo_tempo)
        log.append(f"Peça {peca+1} → Linha {linha}, Máquina {maquina} (vel={vel}) → pronta em t={novo_tempo}")

        # Reinsere máquina com novo tempo de disponibilidade
        heapq.heappush(heap, (novo_tempo, linha, maquina, vel))

    return tempo_final, log

# Exemplo: 2 linhas, cada uma com máquina moderna (vel=2) e antiga (vel=5)
linhas = [[2, 5], [3, 6]]
makespan, log = escalonar_producao(8, linhas)
print(f"Tempo total de produção: {makespan}")
for entrada in log:
    print(entrada)
OperaçãoComplexidadeObservação
Inicialização do heapO(M log M)M = total de máquinas
Por peça produzidaO(log M)pop + push no heap
TotalO(n log M)n peças, M máquinas
🏭 Este algoritmo é a base de escalonadores em sistemas operacionais (CPU scheduling), clusters de computação (Hadoop, Spark) e gerenciamento de filas em serviços de nuvem. O problema em sua forma geral (Job Shop Scheduling) é NP-completo, mas a versão com máquinas independentes tem solução gulosa ótima.
Visualização

Máquinas modernas (verde) recebem mais peças por ficarem disponíveis mais rápido.

Soma e Multiplicação de Matrizes [M×N]

Soma: O(M·N) Multiplicação: O(M·N·P) Espaço: O(M·P) Álgebra Linear Computacional
Enunciado: Implemente operações de soma (A[M×N] + B[M×N]) e multiplicação (A[M×N] × B[N×P]) de matrizes de ordem arbitrária. Analise as pré-condições de cada operação, a complexidade resultante e visualize o padrão de acesso à memória.
Intuição
A soma é elementar: soma-se posição a posição. Exige que A e B tenham exatamente a mesma ordem — qualquer diferença de dimensão torna a operação inválida. O custo é proporcional ao número de elementos: O(M·N).

A multiplicação é mais sutil. Para que A[M×N] × B[N×P] seja definida, o número de colunas de A deve ser igual ao número de linhas de B (ambos N). Cada elemento C[i][j] da matriz resultado é o produto interno (dot product) da linha i de A com a coluna j de B — N multiplicações e N−1 somas. Como há M×P elementos em C, o custo total é O(M·N·P). Para matrizes quadradas n×n, isso colapsa para O(n³) — o custo clássico.
Análise das Operações
OperaçãoPré-condiçãoComplexidadeResultado
Soma A + BA e B com mesma ordem [M×N]O(M·N)Matriz [M×N]
Multiplicação A × BColunas(A) = Linhas(B)O(M·N·P)Matriz [M×P]
Caso quadrado (n×n)A e B com ordem [n×n]O(n³)Matriz [n×n]
Implementação
def somar_matrizes(A, B):
    """
    Pré-condição: len(A) == len(B) e len(A[0]) == len(B[0])
    Complexidade: O(M·N) tempo, O(M·N) espaço (resultado)
    """
    M, N = len(A), len(A[0])
    # Cria matriz resultado inicializada com zeros
    C = [[0] * N for _ in range(M)]

    for i in range(M):        # O(M) linhas
        for j in range(N):    # O(N) colunas → total O(M·N)
            C[i][j] = A[i][j] + B[i][j]
    return C


def multiplicar_matrizes(A, B):
    """
    A[M×N] × B[N×P] → C[M×P]
    Pré-condição: len(A[0]) == len(B)  (colunas de A = linhas de B)
    Complexidade: O(M·N·P) tempo, O(M·P) espaço (resultado)
    """
    M = len(A)
    N = len(A[0])  # = len(B) — dimensão compartilhada
    P = len(B[0])
    C = [[0] * P for _ in range(M)]

    for i in range(M):          # Linhas de A
        for j in range(P):      # Colunas de B
            for k in range(N):  # Produto interno: acumula N termos
                C[i][j] += A[i][k] * B[k][j]
    return C


# Exemplo:
A = [[1, 2, 3],   # A[2×3]
     [4, 5, 6]]

B = [[7, 8],     # B[3×2]
     [9, 10],
     [11, 12]]

C = multiplicar_matrizes(A, B)   # C[2×2]
# C[0][0] = 1·7 + 2·9 + 3·11 = 58
# C[0][1] = 1·8 + 2·10 + 3·12 = 64
print(C)  # [[58, 64], [139, 154]]
⚡ O algoritmo ingênuo O(n³) é o padrão prático. Existem algoritmos assintoticamente melhores: Strassen (O(n^2.807)) e Coppersmith-Winograd (O(n^2.373)), mas com constantes tão altas que só superam o ingênuo para n absurdamente grande. Para uso geral, bibliotecas como NumPy usam BLAS otimizado — mesma complexidade, mas com paralelismo SIMD e cache-oblivious memory layouts.

Torre de Hanói — Recursivo vs. Iterativo

Tempo: O(2ⁿ) Rec: O(n) pilha Iter: O(2ⁿ) heap Recursão / Pilha Explícita
Enunciado: Mova n discos do pino A para o pino C usando B como auxiliar, nunca colocando um disco maior sobre um menor. Implemente as versões recursiva e iterativa, compare os custos, a natureza do problema e a relação com o Big O. Adicione visualização gráfica com OpenGL.
⚠ Nota sobre linguagem e OpenGL: Este é o único problema do repositório implementado em C++. A escolha é deliberada: a visualização gráfica em tempo real exige acesso direto à API OpenGL (via GLUT/GLFW), que não está disponível em Python sem camadas de abstração pesadas. C++ também oferece controle fino sobre a pilha de chamadas — relevante para comparar o uso de memória das duas abordagens. Os pseudocódigos em Python abaixo servem exclusivamente de referência conceitual para a prova escrita.
A Natureza do Problema
Hanói é um problema inerentemente recursivo por definição matemática: para mover n discos de A para C, é necessário e suficiente mover n−1 discos de A para B, mover o disco maior de A para C, e depois mover os n−1 discos de B para C. Não existe atalho — o número mínimo de movimentos é exatamente 2ⁿ − 1, independente da abordagem usada.

O que muda entre recursivo e iterativo não é o número de movimentos, mas sim como o estado é gerenciado: na versão recursiva, a call stack do sistema operacional guarda implicitamente os estados pendentes (O(n) frames). Na iterativa, simulamos essa mesma pilha explicitamente no heap (O(2ⁿ) estados no pior caso).

def hanoi_recursivo(n, origem, destino, auxiliar, movimentos=[]):
    # ─── Caso base ────────────────────────────────────────────────────
    if n == 1:
        movimentos.append((origem, destino))
        return

    # Passo 1: mover n−1 discos superiores do Origem → Auxiliar
    hanoi_recursivo(n - 1, origem, auxiliar, destino, movimentos)

    # Passo 2: mover o disco maior (o de baixo) do Origem → Destino
    movimentos.append((origem, destino))

    # Passo 3: mover n−1 discos do Auxiliar → Destino
    hanoi_recursivo(n - 1, auxiliar, destino, origem, movimentos)

    return movimentos

# Uso: hanoi_recursivo(3, 'A', 'C', 'B')
# Gera exatamente 2³ − 1 = 7 movimentos
# Profundidade máxima da call stack: n frames
def hanoi_iterativo(n, origem, destino, auxiliar):
    """
    Simula a recursão com uma pilha explícita no heap.
    Cada entrada na pilha é um "estado suspenso" equivalente
    a um frame de chamada recursiva não resolvido.
    """
    # Pilha: cada estado é (n, origem, destino, auxiliar)
    pilha = [(n, origem, destino, auxiliar)]
    movimentos = []

    while pilha:
        cn, org, dst, aux = pilha.pop()  # LIFO — último a entrar, primeiro a sair

        if cn == 1:
            # Caso base: movimento atômico
            movimentos.append((org, dst))
            continue

        # Empilhamos na ordem INVERSA para executar na ordem correta:
        # 3° (será executado último): n−1 discos do aux → dst
        pilha.append((cn - 1, aux, dst, org))
        # 2° (será executado no meio): disco maior org → dst
        pilha.append((1, org, dst, aux))
        # 1° (será executado primeiro): n−1 discos de org → aux
        pilha.append((cn - 1, org, aux, dst))

    return movimentos

# Mesmos 2ⁿ−1 movimentos — porém o espaço usado é O(2ⁿ) no heap
# vs O(n) na call stack da versão recursiva.

// A visualização em Canvas desta página é a demonstração equivalente no browser.
// Compile com: g++ -o hanoi hanoi.cpp -lGL -lGLU -lglut

#include <GL/glut.h>
#include <stack>
#include <vector>
#include <tuple>
#include <cmath>

using namespace std;
using Move = pair<int,int>;  // (pino_origem, pino_destino)
struct Estado { int n, org, dst, aux; };

// ─── Versão Recursiva — O(2ⁿ) tempo, O(n) espaço de stack ────────────
void hanoi_rec(int n, int org, int dst, int aux, vector<Move>& moves) {
    if (n == 1) { moves.push_back({org, dst}); return; }
    hanoi_rec(n-1, org, aux, dst, moves);
    moves.push_back({org, dst});
    hanoi_rec(n-1, aux, dst, org, moves);
}

// ─── Versão Iterativa — O(2ⁿ) tempo, O(2ⁿ) espaço de heap ───────────
vector<Move> hanoi_iter(int n, int org, int dst, int aux) {
    stack<Estado> pilha;
    vector<Move> moves;
    pilha.push({n, org, dst, aux});
    while (!pilha.empty()) {
        auto [cn, o, d, a] = pilha.top(); pilha.pop();
        if (cn == 1) { moves.push_back({o, d}); continue; }
        pilha.push({cn-1, a, d, o});
        pilha.push({1,    o, d, a});
        pilha.push({cn-1, o, a, d});
    }
    return moves;
}

// ─── Estado global da animação ────────────────────────────────────────
vector<Move> moves;
int step = 0, N = 5;
vector<int> pinos[3];  // pinos[0]=A, pinos[1]=B, pinos[2]=C

// ─── OpenGL: desenha um disco como quad colorido ──────────────────────
void drawDisk(float cx, float cy, float w, float h,
              float r, float g, float b) {
    glColor3f(r, g, b);
    glBegin(GL_QUADS);
        glVertex2f(cx - w/2, cy);
        glVertex2f(cx + w/2, cy);
        glVertex2f(cx + w/2, cy + h);
        glVertex2f(cx - w/2, cy + h);
    glEnd();
}

// ─── OpenGL: callback de display ──────────────────────────────────────
void display() {
    glClear(GL_COLOR_BUFFER_BIT);
    /* desenha pinos, base e discos usando drawDisk()
       com cores mapeadas pelo tamanho do disco */
    glutSwapBuffers();
}

// ─── Timer: avança um passo da animação a cada 400ms ──────────────────
void timer(int) {
    if (step < (int)moves.size()) {
        auto [from, to] = moves[step++];
        pinos[to].push_back(pinos[from].back());
        pinos[from].pop_back();
        glutPostRedisplay();
    }
    glutTimerFunc(400, timer, 0);
}

int main(int argc, char** argv) {
    for (int i = N; i >= 1; i--) pinos[0].push_back(i);
    hanoi_rec(N, 0, 2, 1, moves);  // ou hanoi_iter()
    glutInit(&argc, argv);
    glutInitDisplayMode(GLUT_DOUBLE | GLUT_RGB);
    glutInitWindowSize(800, 500);
    glutCreateWindow("Torre de Hanói — OpenGL");
    glutDisplayFunc(display);
    glutTimerFunc(400, timer, 0);
    glutMainLoop();
}
